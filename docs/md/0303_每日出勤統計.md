# 每日出勤統計

## 1. 設計目標

將原始打卡記錄彙整為每日出勤摘要，包含上下班時間、工時計算、遲到早退判斷等。這是差勤系統最核心的資料表，也是薪資計算的重要依據。

### 1.1 為什麼需要每日出勤統計

1. **精簡查詢**：原始打卡一天多筆，統計後一天一筆，方便快速查詢
2. **工時計算**：自動計算實際工時、加班、遲到、早退
3. **異常偵測**：辨識缺勤、打卡配對不完整等需人工審核的狀況
4. **薪資依據**：計薪時直接讀取每日統計，不必每次重新分析打卡記錄

### 1.2 資料流程

```
原始打卡表（clock_records）── 一天多筆
    ↓ 每日批次處理（凌晨自動 or 手動觸發）
每日出勤統計（daily_attendances）── 一天一筆（員工 × 日期唯一）
    ↓ 每月彙總
每月出勤統計（monthly_attendances）── 一月一筆
```

## 2. 資料表設計

### daily_attendances

```
daily_attendances
├── id
```

## 3. 時間欄位說明

### 三組時間的差異

| 欄位 | 來源 | 用途 | 範例 |
|------|------|------|------|
| scheduled_in / scheduled_out | 排班或員工預設 | 判斷遲到早退的基準 | 09:00 — 18:00 |
| clocked_in / clocked_out | 原始打卡記錄 | 記錄實際打卡時間，不做調整 | 08:55 — 18:10 |
| work_start / work_end | 依公司政策計算 | 計薪用的起訖時間 | 09:00 — 18:10 |

**範例**：

```
排定時間：09:00 — 18:00
打卡時間：08:55 — 18:10（早到 5 分鐘，晚退 10 分鐘）
工作時間：09:00 — 18:10（不計早到，計入延遲下班）
```

### 工時欄位計算方式

| 欄位 | 計算公式 | 說明 |
|------|----------|------|
| scheduled_minutes | scheduled_out − scheduled_in − break_minutes | 應工作分鐘數 |
| work_minutes | work_end − work_start − break_minutes | 實際工作分鐘數（計薪用） |
| overtime_minutes | work_minutes − scheduled_minutes（取正值） | 加班分鐘數 |
| late_minutes | clocked_in − scheduled_in（超過容許值時） | 遲到分鐘數 |
| early_leave_minutes | scheduled_out − clocked_out（超過容許值時） | 早退分鐘數 |

## 4. 異常標記與分級

### 輕微異常（系統自動處理）

| 標記 | 判斷條件 | 處理方式 | status |
|------|----------|----------|--------|
| is_late | 遲到超過容許時間 | 系統自動扣薪 | approved |
| is_early_leave | 早退超過容許時間 | 系統自動扣薪 | approved |
| is_overtime | 工時超過應工作時數 | 系統自動計算加班費 | approved |

### 嚴重異常（需人工審核）

當 `is_abnormal = true` 時，`status` 設為 `pending`，需 HR 或主管介入：

1. 有上班打卡，無下班打卡
2. 有下班打卡，無上班打卡
3. 有休息開始，無休息結束（或反之）
4. 完全無打卡記錄（缺勤）

**設計理念**：遲到、早退可由系統自動扣薪，不算嚴重異常；只有配對不完整、無法計算工時的情況才標記 `is_abnormal`。

## 5. 審核狀態（status）

| 值 | 說明 | 觸發情境 |
|----|------|----------|
| `pending` | 待審核 | 有嚴重異常（is_abnormal = true） |
| `approved` | 已審核通過 | 打卡正常或輕微異常，自動設定 |
| `adjusted` | 已調整 | HR 手動修改工作時間後重新計算 |
| `voided` | 已作廢 | 整天記錄作廢（如系統錯誤） |

## 6. 核心業務邏輯

### 6.1 每日批次處理流程

每天凌晨自動處理前一天的出勤記錄：

```
1. 取得或建立每日記錄（employee_id + work_date 唯一）
2. 取得該員工當天所有 approved 的打卡記錄（依 clocked_at 排序）
3. 取得排定時間（排班 > 員工預設 > 系統預設）
4. 結合行事曆判斷是否為工作日（見 0301_行事曆管理）
5. 分析打卡記錄：
   - 第一筆 clock_type=in → clocked_in
   - 最後一筆 clock_type=out → clocked_out
   - 檢查配對完整性
6. 計算工作時間（依計薪政策）
7. 判斷遲到、早退、加班
8. 儲存統計結果
```

### 6.2 工作時間計算（計薪政策）

| 情境 | 計薪設定 | work_start | work_end |
|------|----------|------------|----------|
| 早到 | count_early_arrival = false | scheduled_in（不計早到） | — |
| 早到 | count_early_arrival = true | clocked_in（計入早到） | — |
| 延遲下班 | count_late_departure = true | — | clocked_out（計入延遲下班） |
| 延遲下班 | count_late_departure = false | — | scheduled_out（不計延遲下班） |

### 6.3 排程

```
artisan 指令：attendance:process-daily {date?}

預設處理前一天；可帶日期參數重新處理指定日期。

排程：每天凌晨 01:00 自動執行（withoutOverlapping）
```

## 7. 審核與調整

### 審核流程

```
系統處理 → status = pending（嚴重異常）
    ↓
HR 查看異常列表 → 篩選 status=pending & is_abnormal=1
    ↓
├── 審核通過 → status = approved, reviewed_by, reviewed_at
├── 調整時間 → 手動修正 work_start/work_end → 重算工時 → status = adjusted
└── 作廢     → status = voided
```

### 調整後重算工時

HR 修改 `work_start` / `work_end` 後，系統自動重算：

```
work_minutes = work_end − work_start − break_minutes
overtime_minutes = max(0, work_minutes − scheduled_minutes)
```

## 8. Config 設定

```
config/hrm.php

容許時間：
  late_threshold_minutes = 5          -- 容許遲到 5 分鐘
  early_leave_threshold_minutes = 5   -- 容許早退 5 分鐘

預設工作時間：
  default_work_start = 09:00
  default_work_end = 18:00
  default_break_minutes = 60

計薪政策：
  count_early_arrival = false         -- 不計早到
  count_late_departure = true         -- 計入延遲下班
```

## 9. 與其他表的關聯

### 9.1 資料輸入

| 來源 | 提供資訊 |
|------|----------|
| clock_records（見 [0302](0302_原始打卡表.md)） | 原始打卡記錄；第一筆 in → clocked_in，最後一筆 out → clocked_out |
| calendar_days（見 [0301](0301_行事曆管理.md)） | 判斷是否為工作日（day_type） |
| employees | 預設工作時間（default_work_start/end、default_break_minutes） |
| shifts（可選） | 排班時間，優先於員工預設 |

### 9.2 資料輸出

| 目標 | 取用欄位 |
|------|----------|
| monthly_attendances（每月統計） | 彙總 work_minutes、overtime_minutes、late/early_leave 次數等 |
| 薪資計算 | work_minutes → 基本薪資、overtime_minutes → 加班費、late_minutes → 遲到扣款、early_leave_minutes → 早退扣款 |

## 10. 假日出勤

非工作日有打卡記錄時：

- `is_holiday_work = true`
- 全部工時計入 `overtime_minutes`
- 依勞基法計算加班費倍率（依公司政策設定）
