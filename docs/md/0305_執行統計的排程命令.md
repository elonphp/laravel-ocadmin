# 執行統計的排程命令

## 1. 設計目標

提供 Artisan 指令，將原始打卡記錄自動轉換為每日出勤統計，再彙總為每月出勤統計。支援排程自動執行與手動觸發。

### 1.1 資料流程

```
原始打卡表（clock_records）
    ↓ attendance:process-daily
每日出勤統計（daily_attendances）
    ↓ attendance:process-monthly
每月出勤統計（monthly_summaries）
```

### 1.2 相關設定檔

```
config/hrm.php
├── late_threshold_minutes        -- 容許遲到分鐘數（default 5）
├── early_leave_threshold_minutes -- 容許早退分鐘數（default 5）
├── default_work_start            -- 預設上班時間（default 09:00）
├── default_work_end              -- 預設下班時間（default 18:00）
├── default_break_minutes         -- 預設休息分鐘數（default 60）
├── count_early_arrival           -- 計入早到（default false）
└── count_late_departure          -- 計入延遲下班（default true）
```

## 2. 指令一：每日出勤處理

### 2.1 簽名

```
attendance:process-daily {date?} {--month=} {--force}
```

| 參數 | 說明 | 範例 |
|------|------|------|
| `date` | 指定日期（Y-m-d），預設昨天 | `2025-10-01` |
| `--month` | 指定月份（YYYY-MM），處理整月 | `--month=2025-10` |
| `--force` | 強制重算已審核的記錄 | |

### 2.2 處理邏輯

```
1. 解析日期範圍（單日 or 整月）
2. 取得所有在職員工（is_active = true）
3. 從 clock_records（status = approved）取得打卡記錄
4. 按 employee_id + date 分組，逐組處理：
   a. 第一筆 clock_type=in → clocked_in
   b. 最後一筆 clock_type=out → clocked_out
   c. 檢查配對完整性 → 標記異常
   d. 計算 work_start / work_end（依 config 計薪策略）
   e. 計算 work_minutes / overtime / late / early_leave
5. updateOrCreate 寫入 daily_attendances
```

### 2.3 保護機制

- 已 `approved` 或 `adjusted` 的記錄**不覆蓋**（避免破壞人工審核結果）
- 使用 `--force` 可強制重算所有記錄

### 2.4 工時計算公式

| 欄位 | 計算方式 |
|------|----------|
| work_start | count_early_arrival ? clocked_in : max(clocked_in, scheduled_in) |
| work_end | count_late_departure ? clocked_out : min(clocked_out, scheduled_out) |
| work_minutes | work_end − work_start − break_minutes（取正值） |
| overtime_minutes | work_minutes − scheduled_minutes（取正值） |
| late_minutes | clocked_in − scheduled_in（超過 late_threshold 時） |
| early_leave_minutes | scheduled_out − clocked_out（超過 early_leave_threshold 時） |

## 3. 指令二：每月統計處理

### 3.1 簽名

```
attendance:process-monthly {year_month?}
```

| 參數 | 說明 | 範例 |
|------|------|------|
| `year_month` | 指定月份（YYYY-MM），預設上個月 | `2025-10` |

### 3.2 處理邏輯

```
1. 從 daily_attendances 取得該月有記錄的員工
2. 逐位員工彙總：
   a. scheduled_workdays ← COUNT(全部記錄)
   b. actual_workdays    ← COUNT(is_absent=false AND work_minutes>0)
   c. absent_days        ← COUNT(is_absent=true)
   d. holiday_workdays   ← COUNT(is_holiday_work=true)
   e. scheduled_minutes  ← SUM(scheduled_minutes)
   f. work_minutes       ← SUM(work_minutes)
   g. weekday_overtime   ← SUM(overtime_minutes WHERE is_holiday_work=false)
   h. holiday_overtime   ← SUM(work_minutes WHERE is_holiday_work=true)
   i. overtime_minutes   ← weekday_overtime + holiday_overtime
   j. late_count / late_minutes / early_leave_count / early_leave_minutes
3. updateOrCreate 寫入 monthly_summaries
4. 更新 calculated_at
```

### 3.3 狀態判斷

| 條件 | 寫入 status |
|------|-------------|
| 月份已結束 | `approved` |
| 月份尚未結束 | `draft` |
| 已 `locked` | 不更新（跳過） |

## 4. 排程設定

設定於 `routes/console.php`：

| 排程 | 指令 | 時間 | 說明 |
|------|------|------|------|
| 每日出勤 | `attendance:process-daily` | 每天 02:00 | 處理昨天 |
| 月出勤補算 | `attendance:process-daily --month={上月}` | 每月 1 日 02:00 | 重新處理上月整月 |
| 當月月報 | `attendance:process-monthly {當月}` | 每天 03:00 | 更新當月草稿 |
| 上月月報 | `attendance:process-monthly` | 每月 1 日 03:00 | 結算上月（預設） |

所有排程皆設有 `withoutOverlapping()`，防止重複執行。

### 4.1 手動執行

```bash
# 處理指定日期
php artisan attendance:process-daily 2025-10-01

# 處理整月
php artisan attendance:process-daily --month=2025-10

# 強制重算（覆蓋已審核記錄）
php artisan attendance:process-daily --month=2025-10 --force

# 處理指定月份的月統計
php artisan attendance:process-monthly 2025-10

# 處理上個月（預設）
php artisan attendance:process-monthly

# 查看排程清單
php artisan schedule:list
```
