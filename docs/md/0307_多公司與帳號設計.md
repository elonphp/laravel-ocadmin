# 多公司與帳號設計

## 1. 設計目標

建立以**使用者帳號**為核心的多公司存取機制，讓登入者可在授權範圍內切換公司，操作對應公司的人資資料。

### 1.1 設計原則

公司別的存取權限綁定在**使用者（users）**，而非員工（employees）。理由：

| 場景 | 說明 |
|------|------|
| 外部人員需要存取系統 | 會計師、勞動顧問、法律顧問等，非任何公司員工，但需操作系統 |
| 集團總部管理 | 一個帳號需跨公司管理多間子公司的人資資料 |
| 員工調動 | HR 歸屬（`employees.company_id`）與系統存取（`company_user`）獨立，互不干擾 |
| IT 管理員 | 無員工身分，但需要存取所有公司進行系統維護 |

### 1.2 歸屬 vs 存取

兩個不同概念，分開處理：

```
歸屬（HR 關係）：employee.company_id = 2      → 這個人「在星河科技上班」
存取（系統權限）：users.company_ids: [1,2,3]  → 這個帳號「能看到哪些公司的資料」
```

- **歸屬**掛在 `employees.company_id`（員工屬於哪間公司）
- **存取**掛在 `users.company_ids` JSON 欄位（帳號能操作哪些公司）
- 沒有帳號的員工沒有 `company_ids`（不登入系統）
- 沒有員工記錄的帳號也能透過 `company_ids` 存取公司資料

詳見 [0204_Inertia範例-選項項目-集團公司與部門](0204_Inertia範例-選項項目-集團公司與部門.md) 第一部分與第五部分。

## 2. 資料層

### 2.1 users 表新增欄位

```php
// Migration: add company_ids and default_company_id to users
Schema::table('users', function (Blueprint $table) {
    $table->json('company_ids')->nullable()->after('last_name');           // 可存取的公司 ID 列表
    $table->foreignId('default_company_id')->nullable()
          ->after('company_ids')
          ->constrained('hrm_companies')->nullOnDelete();                  // 預設公司
});
```

| 欄位 | 類型 | 說明 |
|------|------|------|
| `company_ids` | JSON (nullable) | 可存取的公司 ID 陣列，如 `[1,2,3]` |
| `default_company_id` | FK (nullable) | 預設公司，登入時自動選用。刪除公司時設為 null |

### 2.2 Model

**User model：**

```php
// app/Models/User.php

protected function casts(): array
{
    return [
        'company_ids' => 'array',
    ];
}

/**
 * 取得使用者可存取的公司（查詢 Collection）
 */
public function accessibleCompanies()
{
    $ids = $this->company_ids ?? [];
    return \App\Models\Hrm\Company::whereIn('id', $ids)->where('is_active', true)->get();
}

public function defaultCompany(): BelongsTo
{
    return $this->belongsTo(\App\Models\Hrm\Company::class, 'default_company_id');
}
```

**反查（某公司有哪些使用者）：**

```php
User::whereJsonContains('company_ids', $companyId)->get();
```

使用頻率極低，效能可接受。

## 3. 登入與公司選定流程

### 3.1 登入流程

```
使用者登入
  ↓
驗證帳號密碼（Auth::attempt）
  ↓
查詢可存取公司列表
  ├── super_admin → 所有啟用中公司（不受 company_ids 限制）
  └── 一般使用者 → users.company_ids 中的公司
  ↓
決定當前公司
  ├── 有 default_company_id 且在可存取範圍 → 使用預設公司
  ├── 無預設或預設不在範圍 → 取列表第一筆
  └── 列表為空 → 不設定（需處理無公司情境）
  ↓
寫入 session('current_company_id')
  ↓
導向後台首頁
```

### 3.2 Session 管理

```php
// 設定當前公司
session(['current_company_id' => $companyId]);

// 取得當前公司 ID（可封裝為 helper）
function currentCompanyId(): ?int
{
    return session('current_company_id');
}
```

## 4. 公司切換

### 4.1 UI 位置

頂端列（header），位於語言選單左側：

```
┌──────────────────────────────────────────────────────┐
│  [星河科技 ▾]    [🌐 zh-Hant ▾]    [👤 使用者 ▾]  登出  │
└──────────────────────────────────────────────────────┘
```

### 4.2 下拉選單內容

- 列出使用者有權存取的所有啟用中公司
- 當前公司顯示打勾或高亮
- `super_admin` 顯示所有啟用中公司
- 集團階層可用縮排顯示：

```
┌─────────────────────┐
│ ✓ 星河科技           │  ← 當前
│   雲端數位           │
│   天行集團（母公司）   │
│ ─────────────────── │
│   晨光創意           │  ← 獨立公司
└─────────────────────┘
```

### 4.3 切換流程

```
使用者點擊其他公司
  ↓
POST /admin/switch-company  { company_id: 3 }
  ↓
後端驗證：company_id 是否在使用者可存取範圍
  ├── 是 → 更新 session('current_company_id') → 重導回前一頁
  └── 否 → 返回 403
```

### 4.4 API 端點

| Method | URL | 說明 |
|--------|-----|------|
| POST | `/{locale}/admin/switch-company` | 切換當前公司 |

Request：

```json
{ "company_id": 3 }
```

後端驗證後更新 session，重導回 `url()->previous()`。

## 5. 中介層 SetCurrentCompany

### 5.1 職責

- 放在路由中介層堆疊中，位於 `auth` 之後
- 確保 session 中有合法的 `current_company_id`
- 若 session 中的公司不在使用者授權範圍，自動修正

### 5.2 中介層堆疊

```
setLocale → auth → accessBackend → setCurrentCompany → logRequest
```

### 5.3 處理邏輯

```php
namespace App\Http\Middleware;

class SetCurrentCompany
{
    public function handle($request, Closure $next)
    {
        $user = auth()->user();

        // 取得可存取公司 ID 列表
        if ($user->hasRole('super_admin')) {
            $companyIds = Company::where('is_active', true)->pluck('id');
        } else {
            $companyIds = collect($user->company_ids ?? []);
        }

        $currentId = session('current_company_id');

        // 驗證當前公司是否合法
        if (!$currentId || !$companyIds->contains($currentId)) {
            // 嘗試預設公司
            $defaultId = $user->default_company_id;
            if ($defaultId && $companyIds->contains($defaultId)) {
                $currentId = $defaultId;
            } else {
                $currentId = $companyIds->first();
            }
            session(['current_company_id' => $currentId]);
        }

        // 將公司列表與當前公司分享給 View
        view()->share('currentCompanyId', $currentId);
        view()->share('switchableCompanies', Company::whereIn('id', $companyIds)->where('is_active', true)->get());

        return $next($request);
    }
}
```

### 5.4 無公司情境

若使用者的可存取公司列表為空（`$companyIds` 為空集合）：

- `current_company_id` 設為 `null`
- 頂端列不顯示公司選單
- HRM 相關查詢回傳空結果（不出錯）

## 6. 查詢範圍

### 6.1 公司過濾原則

所有 HRM 模組的資料查詢，需加上當前公司過濾：

```php
// 員工查詢
Employee::where('company_id', currentCompanyId())->get();

// 每日出勤（透過員工的公司歸屬）
DailyAttendance::whereHas('employee', function ($q) {
    $q->where('company_id', currentCompanyId());
})->get();

// 部門（部門直接屬於公司）
Department::where('company_id', currentCompanyId())->get();
```

### 6.2 super_admin 例外

`super_admin` 仍受 `current_company_id` 過濾。差異在於：

- `super_admin` 的可切換公司列表包含所有啟用中公司（不受 `company_ids` 限制）
- 查詢範圍仍以 `current_company_id` 過濾，確保資料一致性

若需要全公司彙總（如集團報表），由獨立的報表功能處理，不走一般查詢流程。

### 6.3 Global Scope（可選）

可考慮在 Employee model 加上 Global Scope 自動過濾：

```php
// app/Models/Hrm/Employee.php

protected static function booted(): void
{
    static::addGlobalScope('company', function (Builder $builder) {
        if ($companyId = currentCompanyId()) {
            $builder->where('company_id', $companyId);
        }
    });
}
```

> **注意**：Global Scope 會影響所有查詢（含關聯載入），若有跨公司查詢需求，需用 `withoutGlobalScope('company')` 排除。第一版建議先在 Controller 層手動過濾，確認穩定後再考慮 Global Scope。

## 7. 使用者管理介面

### 7.1 使用者編輯頁面新增欄位

在 Ocadmin 的使用者編輯表單中，新增以下欄位：

| 欄位 | UI 元件 | 說明 |
|------|---------|------|
| 可用公司 | 多選下拉（Select2） | 勾選此使用者可存取的公司，寫入 `users.company_ids` JSON |
| 預設公司 | 單選下拉 | 從「可用公司」中選擇一個，寫入 `users.default_company_id` |

### 7.2 操作邏輯

- 「可用公司」變更時，「預設公司」的選項同步更新
- 若移除的公司包含當前預設公司，預設公司自動清空
- `super_admin` 不需設定可用公司（自動擁有所有公司存取權），但仍可設定預設公司

## 8. 存取範例

```
users.company_ids（系統存取權）
├── IT管理員 (user, 非員工)    → [1,2,3,4]（天行集團, 星河科技, 雲端數位, 晨光創意）← 全部
├── 集團HR   (user + 員工)    → [1,2,3]  （天行集團, 星河科技, 雲端數位）         ← 集團內
├── 晨光員工 (user + 員工)     → [4]      （晨光創意）                            ← 只有自家
├── 外部會計師 (user, 非員工)   → [2,3]    （星河科技, 雲端數位）                   ← 服務對象
└── 工讀生   (員工, 無 user)   → 沒有帳號，無 company_ids

登入情境：
├── 集團HR 登入 → default_company_id=2(星河科技) → session 預設星河
│   → 頂端列可切換：天行集團 / 星河科技 / 雲端數位
├── 外部會計師登入 → default_company_id=2 → session 預設星河
│   → 頂端列可切換：星河科技 / 雲端數位
└── 晨光員工登入 → default_company_id=4(晨光) → session 預設晨光
    → 頂端列只有一間，選單可隱藏或顯示為唯讀
```
