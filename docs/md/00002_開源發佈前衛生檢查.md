# 開源發佈前衛生檢查

## 背景

本專案（**LaravelOcadmin**）是 Laravel 基礎範例專案，作為多個衍生系統的基底（帳號管理中心、POS、HRM 等）。規劃將本專案公開發佈至 GitHub。

本文件說明：
- 為什麼開源基礎專案是安全的
- 發佈前必須完成的衛生檢查
- 發佈後 base 與衍生專案各自的安全責任

---

## 為什麼開源是安全的（原則）

真正的安全來自**正確的加密、認證、授權實作**，不是靠「程式碼保密」。參考業界事實：

- **Laravel 本身**：MIT 授權，全部公開
- **Laravel Passport / Sanctum**：公開
- **WordPress / WooCommerce / OpenCart**：全球數百萬正式站運行公開程式碼
- **Symfony、Django、Rails**：全部開源

開源的優勢：
- 更多眼睛檢視 → 更快發現漏洞
- 社群回報與修補 → 修補速度超越閉源
- 程式碼透明 → 使用者與客戶更容易建立信任
- 招聘、技術品牌、履歷加分

**結論**：是否開源不是安全問題。**發佈前的衛生檢查是否做足**才是。

## 發佈前必做的衛生檢查

### 1. Git 歷史秘密掃描

`.env`、私鑰、密碼一旦曾經 commit 過，**後來刪除也沒用**，歷史裡永遠找得到。必須用工具掃完整 Git 歷史：

```bash
# gitleaks (推薦)
gitleaks detect --source . --verbose

# trufflehog (更深度)
trufflehog git file://. --only-verified
```

若發現歷史中有秘密：
- **最簡單解法**：重建 repo（不保留歷史）後再發佈
- **保留歷史的解法**：用 `git filter-repo` 重寫歷史移除敏感檔；但已外洩的 key/secret 必須**全部作廢重發**，因為可能已被任何 clone 過的人取得

### 2. 確認 `.gitignore` 涵蓋

必須被忽略的項目：

```gitignore
# 環境設定
.env
.env.*.local

# Laravel 產出
/vendor
/node_modules
/storage/*.key
/storage/logs/*
/storage/framework/cache/*
/storage/framework/sessions/*
/storage/framework/views/*
/storage/app/public/*

# Passport / Sanctum 金鑰
/storage/oauth-private.key
/storage/oauth-public.key

# 本機工具
.idea/
.vscode/
*.log
*.sqlite
*.sql
*.dump

# OS
Thumbs.db
.DS_Store
```

### 3. Seeder 與範例資料只用假值

- 使用明顯假的帳號：`test@example.com` / `12345678`、`admin@demo.local`
- **絕對不能**有真實員工 email、真實客戶資料、真實公司名稱
- Seeder 產生的資料要能一眼看出是假資料

### 4. 移除所有硬編碼的 credentials

搜尋並清除：

```bash
# 搜尋可疑關鍵字
grep -rE "(password|secret|token|api_key|client_secret)\s*[:=]" \
  --include="*.php" --include="*.md" --include="*.json" .
```

常見漏網：
- README / docs 裡貼的範例 client_id / client_secret（看起來像示範，其實是真的能用）
- 測試檔裡寫死的 API token
- `config/*.php` 裡寫死的預設值（應改為從 `env()` 讀取）

### 5. 金鑰檔案不能入 repo

Passport 金鑰絕不進 repo：

```bash
# 確認未被追蹤
git ls-files storage/ | grep -E 'oauth.*\.key'
# 應無輸出
```

`APP_KEY` 也不能有預設值 —— `.env.example` 裡只留空字串，README 告訴使用者跑 `php artisan key:generate`。

### 6. 安裝指引強制重設機敏設定

README 第一段必須包含：

```bash
cp .env.example .env
php artisan key:generate              # APP_KEY
php artisan passport:keys --force     # OAuth 金鑰
php artisan migrate --seed
# 登入後立即更改預設 admin 密碼
```

### 7. `composer.json` / `package.json` 檢查

- 所有套件版本鎖定為 stable
- 跑 `composer audit` 與 `npm audit` 確認無已知 CVE
- 若必須用 dev 版套件，在 README 說明原因

### 8. 錯誤訊息不洩漏環境資訊

- 正式環境 `APP_DEBUG=false`（`.env.example` 就設為 `false`）
- 客製化錯誤頁面不顯示 Laravel 版本、stack trace、檔案路徑

## 發佈後的責任分工

### Base 專案（本專案）

- **建立 Security Advisory 流程**：GitHub Security Advisories 或 `SECURITY.md`，接收社群漏洞回報
- **Release Notes 標示 security fix**：讓衍生專案能快速辨識需要緊急合併的版本
- **版本標籤明確**：遵守 SemVer，重大安全修補至少升 patch 版
- **README 明確聲明**：fork 建立新系統的人應先讀 `00002_開源發佈前衛生檢查.md`

### 衍生專案（帳號中心、POS、HRM 等）

- **secret 只存在衍生專案的 `.env`**，base 永遠不碰真實 secret
- **衍生專案的 repo 若不公開**，仍要定期跑 gitleaks —— 內部 repo 也可能因備份外洩、權限設錯而暴露
- **追蹤 base 的 security patch**：訂閱 Release 通知，建立「收到安全版 → X 天內合併上線」的 SLA
- **文件中不貼真實 credentials**：只寫「請查 DB 或由管理員在後台產生」

## 輕度指紋化（減輕自動掃描）

這些**不能擋住認真的攻擊者**，但能過濾 95% 的無差別自動掃描：

- 後台路徑不使用預設 `/admin`（本專案用 `/sys-admin`）
- 移除 `X-Powered-By`、`Server` 等洩漏版本的 header
- 不在公開頁面顯示 Laravel 版本、PHP 版本
- 登入錯誤訊息不區分「帳號不存在」與「密碼錯誤」

## 發佈前最終 checklist

逐項確認（建議發佈當天再跑一次）：

- [ ] `gitleaks detect` 掃描無洩漏
- [ ] `trufflehog git file://.` 掃描無洩漏
- [ ] `.gitignore` 涵蓋 `.env`、`storage/*.key`、`storage/logs/`、`vendor/`、`node_modules/`
- [ ] `git ls-files` 確認無 `.env`、無 `oauth-*.key`
- [ ] Seeder 全部為假資料
- [ ] 無硬編碼 credentials（含 md 文件）
- [ ] `composer audit` 與 `npm audit` 無高風險 CVE
- [ ] `.env.example` 僅保留結構，無真實值
- [ ] README 含安裝強制重設步驟（`key:generate`、`passport:keys --force`）
- [ ] `APP_DEBUG=false` 為 `.env.example` 預設
- [ ] `SECURITY.md` 或 GitHub Security Advisories 已設定
- [ ] `LICENSE` 已選定（建議 MIT，與 Laravel 一致）

## 推薦工具

| 工具 | 用途 |
|------|------|
| [gitleaks](https://github.com/gitleaks/gitleaks) | Git 歷史秘密掃描 |
| [trufflehog](https://github.com/trufflesecurity/trufflehog) | 深度秘密掃描（含驗證） |
| [git-filter-repo](https://github.com/newren/git-filter-repo) | 歷史重寫（移除誤 commit 的秘密） |
| `composer audit` | Composer 套件 CVE 檢查 |
| `npm audit` | npm 套件 CVE 檢查 |
| [Enlightn](https://www.laravel-enlightn.com/) | Laravel 專用安全／效能掃描 |
