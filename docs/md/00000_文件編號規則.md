# 文件編號規則

本目錄下的文件以 5 位數編號命名，前 2 碼為分類，後 3 碼為序號。

本專案為 **LaravelOcadmin 基礎範例專案（`branchMain`，開源版）**，衍生自本專案的具體系統（`branchHuabing`、`branchSunline` 等）遵循相同編號規則，業務段號（11~18xxx）的語意保持跨 branch 一致。

---

## 編號分類

| 編號範圍 | 分類 | 說明 |
|----------|------|------|
| 00xxx | 專案開發規範 | 文件規則、編碼慣例、命名規範、資料庫 Schema 同步、Migration 編號規範、開源發佈檢查等後設文件 |
| 10xxx | 系統全域架構 | 系統架構、Portal、多語/詞彙、認證/權限/安全/帳號、選單、日誌/例外/JSON 回應、Queue、資料架構（EAV/ZTM、Model 分層）、圖片/資源管理等**全系統共通的技術機制** |
| 11xxx | 基礎主檔與組織 | 集團公司、部門、外部組織、行政區、供應商等**跨模組共用的參考資料** |
| 12xxx | 商品型錄 | 商品、分類、選項、BOM、庫存單位等（branchMain 當前無文件） |
| 13xxx | 訂單 | 訂單、訂單明細、訂單狀態、付款等（branchMain 當前無文件） |
| 14xxx | 會員 | 會員資料、點數、消費記錄等（branchMain 當前無文件） |
| 15xxx | 人資與出勤 HRM | 員工、部門、出勤、排班、薪資、勞健保等（branchMain 當前無文件） |
| 16xxx | 報表與統計 | 銷售、庫存、會員分析等報表（branchMain 當前無文件） |
| 17xxx | 系統設定 | 參數設定、Settings 多層擴充、通知、排程等 |
| 18xxx | CRM 客戶關係 | 客戶服務案件、行銷活動等（branchMain 當前無文件） |
| **19xxx** | **前端架構** | **Inertia / React / Livewire / Blade / UI 策略** —— Laravel 傳統後端向前端的延伸層（branchMain 當前無文件） |
| 90xxx | 議題探討與參考 | 尚未實作的提案、技術選型評估、基礎設施備忘、舊版保留參考等 |

---

## 命名格式

```
{編號}_{標題}.md
```

範例：
- `00000_文件編號規則.md`
- `00001_README.md`
- `10000_系統架構.md`
- `10004_認證機制Sanctum.md`
- `17000_參數設定.md`

---

## 序號配置建議

同一分類下，`x000` 通常為該分類的總覽／概述，`x001` 之後為細節主題；但並非硬性規定，實際依內容重要性與閱讀順序安排。

---

## 10xxx 主題群內部排列

10xxx 為最大段（21 檔），內部依主題群彙整、序號大致遞增：

| 序號範圍 | 主題群 |
|----------|--------|
| 10000 | 總覽 |
| 10001 | Portal |
| 10002–10003 | 國際化（多語、詞彙） |
| 10004–10010 | 認證、安全、帳號（Sanctum、使用者記錄、API 安全、權限、2FA、裝置、全域帳號） |
| 10011 | 選單 UI |
| 10012–10014 | 觀測與錯誤處理（日誌、例外、JSON 回應） |
| 10015 | 非同步與排程（Queue） |
| 10016–10018 | 資料架構（Model 分層、EAV/ZTM、多層選項連動） |
| 10019–10020 | 資源管理（圖片、靜態資源快取） |

---

## 跨 branch 一致性

本編號規則與 `branchHuabing`、`branchSunline` 等衍生專案**段號語意保持一致**，差異僅在於：

- 各 branch 的具體檔案集合不同（視業務需要取用）
- 衍生 branch 可能**新增 branch 專屬段**（如 POS 餐飲專屬機制），但通常仍落在 10~18 的業務語意範圍內，或使用 19/90 段

**原則**：段號語意 = 共用規範；檔案內容 = branch 自行決定。
