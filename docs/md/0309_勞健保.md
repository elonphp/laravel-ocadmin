# 勞健保與退休金

## 1. 設計目標

建立勞工保險、全民健保及勞工退休金的投保級距管理與保費自動計算機制，取代現行手動輸入員工保費金額的做法。

### 1.1 解決的問題

| 問題 | 說明 |
|------|------|
| 手動出錯 | HR 需逐一查表輸入每人保費，員工多時容易出錯 |
| 調薪未連動 | 員工調薪後投保級距可能隨之變動，手動作業常遺漏 |
| 政策變動 | 政府每年（或不定期）調整費率、級距、基本工資，需快速全面更新 |
| 眷屬異動 | 健保眷屬人數變動影響保費，需有地方記錄並自動帶入計算 |

### 1.2 資料流程

```
薪資項目定義（is_insurable 標記哪些項目計入投保薪資）
    +
員工薪資配置（各項金額）
    ↓ 加總 is_insurable = true 的項目
月投保薪資基數（≈ 員工月薪資總額）
    ↓ 查表
投保級距表（hrm_insurance_grades）→ 取得「月投保薪資」
    ↓ 套用費率
費率設定（hrm_insurance_rates）→ 月投保薪資 × 費率 × 負擔比例
    ↓
保費金額 → 寫入 payroll_lines
```

### 1.3 與 0308 薪資項目的關係

在 [0308](0308_薪資項目.md) 中，`labor_insurance` 和 `health_insurance` 定義為 `scope = individual`（手動輸入金額）。本文件的目標是將其升級為 `scope = auto`，由系統自動從投保級距和費率計算保費。


## 2. 台灣勞健保制度概覽

### 2.1 四種社會保險

本系統需處理以下四種保險，雇主均有法定義務：

| 項目 | 全稱 | 主管機關 | 計費基礎 |
|------|------|----------|----------|
| 勞保 | 勞工保險（普通事故） | 勞動部勞保局 | 勞保投保薪資 |
| 就保 | 就業保險 | 勞動部勞保局 | 同勞保投保薪資 |
| 健保 | 全民健康保險 | 衛福部健保署 | 健保投保金額 |
| 勞退 | 勞工退休金（新制） | 勞動部勞保局 | 勞退月提繳工資 |

> 職業災害保險（職災保險）自 2022/5/1 獨立為《勞工職業災害保險及保護法》，費率依行業別由政府核定，100% 雇主負擔。本系統暫以固定費率設定處理。

### 2.2 費率與負擔比例（2026 年度參考值）

| 險種 | 費率 | 員工 | 雇主 | 政府 |
|------|------|------|------|------|
| 勞保（普通事故） | 12.5% | 20% | 70% | 10% |
| 就業保險 | 1% | 20% | 70% | 10% |
| 職災保險 | 依行業別（約 0.04%~0.92%） | 0% | 100% | 0% |
| 健保（一般保費） | 5.17% | 30% | 60% | 10% |
| 勞退（雇主提繳） | 6% | — | 100% | — |
| 勞退（自願提繳） | 0~6% | 100% | — | — |

> **重要**：費率會隨政策調整。例如勞保費率從 2025 年的 12% 調升為 2026 年的 12.5%。系統須以 `effective_date` 管理歷次費率。

### 2.3 投保級距制度

台灣的社會保險**不是**直接以實際薪資乘費率，而是：

1. 將薪資**對照級距表**，找到對應等級
2. 以該等級的「月投保薪資」作為計算基礎
3. 月投保薪資 × 費率 × 負擔比例 = 保費

**各險種級距範圍（2026 年）**：

| 險種 | 級數 | 最低 | 最高 |
|------|------|------|------|
| 勞保 | 11 級 | 29,500 | 45,800 |
| 健保 | 58 級 | 29,500 | 313,000 |
| 勞退 | 62 級 | 1,500 | 150,000 |

> 勞保與健保的前 11 級投保薪資相同，但健保級距延伸至 313,000 元。

### 2.4 健保眷屬規則

- 員工可將無職業的配偶、直系尊親屬、未成年子女加入健保
- 眷屬保費由被保險人負擔
- **眷屬人數上限**：超過 3 口以 3 口計（即員工本人 + 最多 3 名眷屬 = 最多按 4 人計費）
- 計算公式：`投保金額 × 5.17% × 30% × (1 + 眷屬人數)`

### 2.5 保費計算公式彙整

**勞保（員工自付）**：
```
月投保薪資 × 12.5% × 20%
```

**就保（員工自付）**：
```
月投保薪資 × 1% × 20%
```

**健保（員工自付）**：
```
投保金額 × 5.17% × 30% × (1 + min(眷屬人數, 3))
```

**勞退（雇主提繳）**：
```
月提繳工資 × 6%
```

**勞退（員工自願提繳）**：
```
月提繳工資 × 自願提繳比例（0~6%）
```

> 實際扣繳時，勞保 + 就保通常合併為一筆「勞保費」，使用同一投保薪資。


## 3. 資料表設計

### 3.1 hrm_insurance_grades（投保級距表）

統一存放勞保、健保、勞退三種級距。結構相同，以 `type` 區分。

```
hrm_insurance_grades
├── id
├── type                    -- labor / health / pension（見 §4.1）
├── effective_date          -- 施行日期（如 2026-01-01）
├── grade_level             -- 等級序號（1, 2, 3...）
├── salary_min              -- 月薪資總額下限（unsigned int）
├── salary_max              -- 月薪資總額上限（unsigned int, nullable; null = 無上限）
├── insured_salary          -- 月投保薪資（unsigned int）
├── timestamps

UNIQUE(type, effective_date, grade_level)
INDEX(type, effective_date)
```

**每當政府公布新級距表**（通常隨基本工資調整，每年 1/1），新增一批記錄即可，歷史資料保留不刪。

### 3.2 hrm_insurance_rates（費率設定表）

```
hrm_insurance_rates
├── id
├── code                    -- 費率代碼（見 §4.2）
├── name                    -- 名稱（如「勞保普通事故」）
├── effective_date          -- 施行日期
├── rate                    -- 費率（decimal 7,4; 如 12.5000 表示 12.5%）
├── employee_ratio          -- 員工負擔比例（decimal 5,4; 如 0.2000）
├── employer_ratio          -- 雇主負擔比例（decimal 5,4; 如 0.7000）
├── government_ratio        -- 政府負擔比例（decimal 5,4; 如 0.1000）
├── note                    -- 備註（如「115年度適用」）
├── timestamps

UNIQUE(code, effective_date)
INDEX(code)
```

### 3.3 Employee 擴充欄位

在 `hrm_employees` 新增保險相關欄位：

```diff
hrm_employees
+ insurance_dependents       -- 健保眷屬人數（tinyint unsigned, default 0, max 3）
+ voluntary_pension_rate     -- 勞退自願提繳比例（decimal 3,1; 0.0~6.0, default 0.0）
```

> `insurance_dependents` 上限 3（超過 3 口以 3 口計）。`voluntary_pension_rate` 為勞退自願提繳比例（0%~6%），多數員工為 0。

| code | 名稱 | 2026 費率 | 員工 | 雇主 | 政府 |

## 5. 計算邏輯

### 5.1 投保薪資基數的決定

```
投保薪資基數 = Σ (is_insurable = true 的薪資項目金額)
```

從 `hrm_employee_salary_items` 取得員工所有 `is_insurable = true` 的項目加總：
- 通常包含：本薪、職務津貼、主管津貼、技術津貼等**經常性給與**
- 通常不含：加班費、年終獎金、差旅費等**非經常性給與**

> `is_insurable` 標記已在 [0308](0308_薪資項目.md) 的 `hrm_salary_items` 中定義。

### 5.2 級距對應（Grade Lookup）

```
function lookupInsuredSalary(string $type, int $monthlySalary, string $date): int
{
    return InsuranceGrade::where('type', $type)
        ->where('effective_date', '<=', $date)
        ->where('salary_min', '<=', $monthlySalary)
        ->where(fn($q) => $q->whereNull('salary_max')
                            ->orWhere('salary_max', '>=', $monthlySalary))
        ->orderByDesc('effective_date')
        ->orderBy('grade_level')
        ->first()
        ->insured_salary;
}
```

**查詢邏輯**：
1. 篩選 `type` 和**最近的施行日**（`effective_date <= 計算日`）
2. 找到薪資落入的區間（`salary_min <= 薪資 <= salary_max`）
3. 取得該等級的 `insured_salary`

### 5.3 保費計算

**勞保費（員工自付）**：
```
勞保投保薪資 × (勞保費率 + 就保費率) × 員工負擔比例
= insured_salary × (12.5% + 1%) × 20%
```

**健保費（員工自付）**：
```
健保投保金額 × 健保費率 × 員工負擔比例 × (1 + 眷屬人數)
= insured_salary × 5.17% × 30% × (1 + min(dependents, 3))
```

**勞退提繳（雇主）**：
```
勞退月提繳工資 × 6%
```

**勞退自願提繳（員工自付）**：
```
勞退月提繳工資 × 自願提繳比例
```

### 5.4 四捨五入規則

- 保費計算結果以**元**為單位，採**四捨五入**至整數
- 勞保局與健保署的級距表中已提供各級的實際負擔金額（可做交叉驗證）


## 6. 與薪資系統的整合

### 6.1 薪資項目定義變更

將 `labor_insurance` 和 `health_insurance` 從 `individual` 升級為 `auto`：

```diff
  labor_insurance:
-   scope: individual
+   scope: auto
+   calc_config: { "method": "insurance", "codes": ["labor", "employment"], "party": "employee" }

  health_insurance:
-   scope: individual
+   scope: auto
+   calc_config: { "method": "insurance", "codes": ["health"], "party": "employee" }
```

新增勞退自願提繳項目（如有需要）：

```
  voluntary_pension:
    scope: auto
    calc_config: { "method": "insurance", "codes": ["pension"], "party": "employee_voluntary" }
```

### 6.2 calc_config 格式

```json
{
  "method": "insurance",
  "codes": ["labor", "employment"],
  "party": "employee"
}
```

| 欄位 | 說明 |
|------|------|
| `method` | 固定為 `insurance`，對應 `InsuranceCalculator` |
| `codes` | 險種代碼陣列（對應 `hrm_insurance_rates.code`） |
| `party` | `employee`（員工負擔）/ `employer`（雇主負擔）/ `employee_voluntary`（自願提繳） |

### 6.3 InsuranceCalculator

新增一個 Calculator，遵循既有 Strategy Pattern：

```php
// Services/Calculators/InsuranceCalculator.php

class InsuranceCalculator implements ItemCalculatorInterface
{
    public function calculate(
        object $calcConfig,
        Employee $employee,
        MonthlySummary $summary,
        array $context = [],
    ): CalculationResult {

        $insurableSalary = $context['insurable_salary'];  // 從 context 取得
        $date = $summary->year_month . '-01';
        $totalAmount = 0;
        $notes = [];

        foreach ($calcConfig->codes as $code) {
            $rate = InsuranceRate::getCurrentRate($code, $date);
            $gradeType = $this->mapCodeToGradeType($code);
            $insuredSalary = InsuranceGrade::lookup($gradeType, $insurableSalary, $date);

            $ratio = match ($calcConfig->party) {
                'employee'           => $rate->employee_ratio,
                'employer'           => $rate->employer_ratio,
                'employee_voluntary' => $employee->voluntary_pension_rate / 100,
            };

            $amount = round($insuredSalary * ($rate->rate / 100) * $ratio);

            // 健保需乘眷屬係數
            if ($code === 'health' && $calcConfig->party === 'employee') {
                $dependents = min($employee->insurance_dependents, 3);
                $amount *= (1 + $dependents);
            }

            $totalAmount += $amount;
            $notes[] = "{$rate->name} {$insuredSalary}×{$rate->rate}%×{$ratio}";
        }

        return new CalculationResult(
            amount: (int) $totalAmount,
            note: implode('；', $notes),
        );
    }

    private function mapCodeToGradeType(string $code): string
    {
        return match ($code) {
            'labor', 'employment', 'accident' => 'labor',
            'health'  => 'health',
            'pension' => 'pension',
        };
    }
}
```

**設計說明：一個 Calculator 處理所有險種**

`InsuranceCalculator` 是通用的，勞保和健保都走同一個 Calculator，差異由 `calc_config.codes` 驅動：

| 薪資項目 | calc_config.codes | 實際計算 |
|---------|-------------------|---------|
| 勞保費 | `["labor", "employment"]` | 分別查勞保費率 12.5% 和就保費率 1%，各自算完加總 |
| 健保費 | `["health"]` | 查健保費率 5.17%，乘眷屬係數 |
| 勞退自願提繳 | `["pension"]` | 查勞退級距，乘自願比例 |

核心的 `foreach ($calcConfig->codes as $code)` 迴圈對每個險種做通用計算：查費率 → 查級距 → `投保薪資 × 費率 × 負擔比例`。勞保因為不需要特殊處理，直接走通用公式即可。唯一的 `if ($code === 'health')` 是因為健保獨有「眷屬加乘」邏輯。

這樣設計的好處：新增險種（如補充保費）時只需定義新的 `calc_config`，不需要新增 Calculator class。

### 6.4 SalaryCalculatorService 擴充

在 `$calculators` 陣列新增：

```php
protected array $calculators = [
    'conditional'   => ConditionalCalculator::class,
    'per_unit'      => PerUnitCalculator::class,
    'daily_rate'    => DailyRateCalculator::class,
    'late_penalty'  => LatePenaltyCalculator::class,
    'overtime'      => OvertimeCalculator::class,
    'insurance'     => InsuranceCalculator::class,    // 新增
];
```

在 `$context` 陣列新增 `insurable_salary`：

```php
$context = [
    'base_salary'       => $baseSalary,
    'daily_rate'        => intdiv($baseSalary, 30),
    'hourly_rate'       => intdiv($baseSalary, 240),
    'insurable_salary'  => $this->calculateInsurableSalary($employee),  // 新增
];
```

`calculateInsurableSalary()` 加總員工所有 `is_insurable = true` 且在效期內的薪資項目金額。


## 7. 政策變動更新機制

### 7.1 變動類型與應對方式

| 變動類型 | 頻率 | 誰來做 | 怎麼做 |
|---------|------|--------|--------|
| **級距表更新** | 通常每年（隨基本工資調整） | HR 或系統管理員 | 匯入新級距（Artisan 指令或管理介面） |
| **費率調整** | 不定期（政府公告） | HR 或系統管理員 | 新增一筆費率記錄（管理介面） |
| **眷屬人數變動** | 隨時（員工申請） | HR | 更新員工資料 |
| **計算邏輯變更** | 罕見（法規重大修正） | 工程師 | 修改 InsuranceCalculator |

### 7.2 級距表更新流程

**方式一：Artisan 指令匯入**

```bash
php artisan insurance:import-grades {type} {effective_date} {file}
```

從 CSV/JSON 檔匯入新的級距表。指令邏輯：
1. 驗證 `type` 和 `effective_date`
2. 檢查是否已有同 `type + effective_date` 的資料（防重複匯入）
3. 批次寫入 `hrm_insurance_grades`
4. 顯示匯入筆數

CSV 格式範例：

```csv
grade_level,salary_min,salary_max,insured_salary
1,0,29500,29500
2,29501,30300,30300
3,30301,31800,31800
...
```

**方式二：管理介面手動輸入**

提供「投保級距管理」頁面，HR 可在介面上新增/檢視各年度級距表。

### 7.3 費率更新流程

在「費率設定」管理介面新增一筆記錄：

1. 選擇險種（code）
2. 填入施行日期（effective_date）
3. 填入新費率（rate）
4. 負擔比例通常不變，如有變動一併更新

系統計算時自動取 `effective_date <= 計算月份` 的最新費率。

### 7.4 歷史資料保護

- **級距表**：不刪除歷史記錄。以 `effective_date` 區分版本。
- **費率**：不刪除歷史記錄。以 `effective_date` 區分版本。
- **薪資單快照**：`hrm_payroll_lines` 已有 `amount` + `note` 快照，即使日後費率或級距變動，歷史薪資單不受影響。

### 7.5 年度更新檢查清單

每年底（或政府公告後），依以下清單更新：

```
□ 1. 確認新年度基本工資是否調整
□ 2. 匯入勞保投保薪資分級表（type=labor）
□ 3. 匯入健保投保金額分級表（type=health）
□ 4. 匯入勞退月提繳工資分級表（type=pension）
□ 5. 確認勞保費率是否調整 → 如有，新增費率記錄
□ 6. 確認就保費率是否調整 → 如有，新增費率記錄
□ 7. 確認健保費率是否調整 → 如有，新增費率記錄
□ 8. 確認職災費率是否調整 → 如有，新增費率記錄
□ 9. 試算數名員工保費，與政府公告負擔金額表交叉驗證
□ 10. 通知 HR 確認完成
```

### 9.3 Model 設計

```php
// InsuranceGrade.php
class InsuranceGrade extends Model
{
    protected $table = 'hrm_insurance_grades';

    /**
     * 根據薪資查找投保薪資。
     */
    public static function lookup(string $type, int $salary, string $date): int
    {
        $grade = static::where('type', $type)
            ->where('effective_date', '<=', $date)
            ->orderByDesc('effective_date')
            ->get()
            ->first(fn ($g) => $salary >= $g->salary_min
                && ($g->salary_max === null || $salary <= $g->salary_max));

        return $grade?->insured_salary
            ?? throw new \RuntimeException("找不到 {$type} 級距（薪資 {$salary}，日期 {$date}）");
    }
}
```

```php
// InsuranceRate.php
class InsuranceRate extends Model
{
    protected $table = 'hrm_insurance_rates';

    /**
     * 取得指定險種在指定日期的有效費率。
     */
    public static function getCurrentRate(string $code, string $date): static
    {
        return static::where('code', $code)
            ->where('effective_date', '<=', $date)
            ->orderByDesc('effective_date')
            ->firstOrFail();
    }
}
```

## 11. 驗證方式

### 11.1 保費交叉驗證

以 2026 年級距為例，月薪 35,000 元的員工（0 眷屬）：

```
勞保投保薪資 = 36,300（第 6 級，34,801~36,300）
健保投保金額 = 36,300（同上）

勞保費（員工）= 36,300 × 12.5% × 20% = 908（四捨五入）
就保費（員工）= 36,300 × 1% × 20%   = 73（四捨五入）
→ 合計勞保費（員工自付）= 981

健保費（員工）= 36,300 × 5.17% × 30% × 1 = 563（四捨五入）
```

可與政府公布的「保險費負擔金額表」比對。

### 11.2 級距邊界測試

- 薪資恰好等於級距邊界值（如 29,500、29,501）
- 薪資超過最高級距（如勞保 > 45,800 應對應 45,800）
- 薪資低於最低級距（如工讀 < 29,500 應對應 29,500）
- 不同 `effective_date` 的級距切換
