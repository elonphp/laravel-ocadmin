# 拖放員工排班

## 1. 設計目標

提供視覺化的排班介面，讓管理者用拖放操作快速指派員工每日班表。

### 1.1 核心需求

- **七日檢視**：顯示一週的排班，左右切換週次，不需要縱軸時間列
- **拖放排班**：左側員工列表可拖放到日期格，自動帶入預設上下班時間
- **即時編輯**：點擊已排的班次可修改當天 scheduled_in / scheduled_out
- **刪除班次**：可移除已排定的班
- **不新增資料表**：直接寫入 `hrm_daily_attendances` 的 `scheduled_in` / `scheduled_out`

### 1.2 資料流程

```
拖放指派 → daily_attendances.scheduled_in / scheduled_out
    ↓ 員工當天打卡
clock_records 產生上下班記錄
    ↓ 凌晨批次 attendance:process-daily
daily_attendances 填入 clocked_in / clocked_out / work_minutes / ...
```

排班建立排定時間，出勤處理指令填入實際打卡與計算結果，兩者互不衝突。

## 2. 畫面設計

```
┌─────────────────────────────────────────────────────────────────┐
│  排班管理                                      ◀ 本週 ▶  今天  │
├────────┬────────────────────────────────────────────────────────┤
│ 員工    │  2/17(一)  │  2/18(二)  │  2/19(三)  │  ...  │ 2/23(日) │
├────────┼────────────┼────────────┼────────────┼───────┼──────────┤
│ 王小明  │ 王小明      │ 王小明      │            │       │          │
│ 李小花  │ 09:00-18:00│ 09:00-18:00│            │       │          │
│ 陳大文  │ 李小花      │            │ 陳大文      │       │          │
│ ...    │ 09:00-18:00│            │ 07:00-14:00│       │          │
│        │            │            │            │       │          │
└────────┴────────────┴────────────┴────────────┴───────┴──────────┘
```

### 2.1 左側員工列表

- 寬度固定 120px，只顯示姓名
- 所有在職員工，依工號排序
- 每個員工是可拖放元素（FullCalendar External Event）
- 拖放後員工不從列表消失（同一人可排多天）

### 2.2 七日日曆區

- 使用 FullCalendar 6 的 `dayGridWeek` 視圖（純日期格，無時間軸）
- 每筆班次顯示：「姓名 HH:MM-HH:MM」，例如「王小明 09:00-18:00」
- 點擊班次 → 彈出 Bootstrap Modal 編輯 scheduled_in / scheduled_out
- 上方工具列：◀ 上週 / 下週 ▶、今天按鈕

### 2.3 班次顯示格式

每筆事件在日曆中顯示為一行文字：

```
王小明 09:00-18:00
```

顏色統一使用 Bootstrap primary，或依部門區分（第一版先統一色）。

## 3. 技術方案

### 3.1 前端函式庫

透過 CDN 引入 FullCalendar 6，不安裝到 vendor：

```html
<!-- FullCalendar 6 -->
<link href="https://cdn.jsdelivr.net/npm/fullcalendar@6.1.15/index.global.min.css" rel="stylesheet">
<script src="https://cdn.jsdelivr.net/npm/fullcalendar@6.1.15/index.global.min.js"></script>
```

FullCalendar 6 的 `index.global.min.js` 已包含 dayGrid + interaction（拖放）插件。

### 3.2 API 端點

| Method | URL | 說明 |
|--------|-----|------|
| GET | `/hrm/schedule` | 排班頁面 |
| GET | `/hrm/schedule/events?start=&end=` | 取得排班事件（JSON） |
| POST | `/hrm/schedule/assign` | 拖放指派 |
| PUT | `/hrm/schedule/update/{id}` | 修改 scheduled_in/out |
| DELETE | `/hrm/schedule/remove/{id}` | 刪除排班 |

### 3.3 events API 回傳格式

```json
[
    {
        "id": 123,
        "title": "王小明 09:00-18:00",
        "start": "2026-02-17",
        "extendedProps": {
            "employee_id": 2,
            "employee_name": "王小明",
            "scheduled_in": "09:00",
            "scheduled_out": "18:00"
        }
    }
]
```

FullCalendar 的 `events` 用 JSON feed，帶 `start` / `end` query 參數自動請求當週資料。

### 3.4 assign API

```
POST /hrm/schedule/assign
{
    "employee_id": 2,
    "date": "2026-02-17"
}
```

後端邏輯：
1. 查找 employee，取 `default_work_start` / `default_work_end`
2. `DailyAttendance::updateOrCreate` 寫入 `scheduled_in` / `scheduled_out`
3. 若該員工當天已有記錄，更新排定時間（不影響已有的打卡資料）
4. 回傳新建 / 更新的事件 JSON

### 3.5 update API

```
PUT /hrm/schedule/update/{dailyAttendanceId}
{
    "scheduled_in": "08:00",
    "scheduled_out": "17:00"
}
```

### 3.6 remove API

```
DELETE /hrm/schedule/remove/{dailyAttendanceId}
```

將 `scheduled_in` / `scheduled_out` 設為 null。若該記錄沒有任何打卡資料（clocked_in/out 都是 null），則直接刪除整筆記錄。

## 4. 後端實作

### 4.1 Controller

```
app/Portals/Ocadmin/Modules/Hrm/Schedule/ScheduleController.php
```

```php
class ScheduleController extends OcadminController
{
    setLangFiles() → ['hrm/schedule']

    index()         → 排班頁面，傳入在職員工列表
    events(Request) → JSON feed，查 daily_attendances WHERE scheduled_in IS NOT NULL
    assign(Request) → 拖放指派，updateOrCreate
    update(Request, DailyAttendance) → 修改 scheduled_in/out
    remove(DailyAttendance)          → 清除排班或刪除記錄
}
```

### 4.2 events 查詢

```php
DailyAttendance::with('employee')
    ->whereNotNull('scheduled_in')
    ->whereBetween('work_date', [$start, $end])
    ->get()
    ->map(fn ($r) => [
        'id'    => $r->id,
        'title' => $r->employee->full_name . ' '
                 . $r->scheduled_in->format('H:i') . '-'
                 . $r->scheduled_out->format('H:i'),
        'start' => $r->work_date->format('Y-m-d'),
        'extendedProps' => [
            'employee_id'   => $r->employee_id,
            'employee_name' => $r->employee->full_name,
            'scheduled_in'  => $r->scheduled_in->format('H:i'),
            'scheduled_out' => $r->scheduled_out->format('H:i'),
        ],
    ]);
```

## 5. 前端實作

### 5.1 FullCalendar 初始化

```js
var calendar = new FullCalendar.Calendar(calendarEl, {
    initialView: 'dayGridWeek',
    locale: 'zh-tw',
    headerToolbar: {
        left: 'prev,next today',
        center: 'title',
        right: ''
    },
    editable: false,
    droppable: true,
    events: { url: eventsUrl, method: 'GET' },
    drop: function(info) { /* POST assign */ },
    eventClick: function(info) { /* 開啟 Modal 編輯 */ },
});
```

### 5.2 External Draggable

```js
new FullCalendar.Draggable(employeeListEl, {
    itemSelector: '.employee-item',
    eventData: function(el) {
        return {
            title: el.dataset.name,
            extendedProps: { employee_id: el.dataset.id }
        };
    }
});
```

### 5.3 編輯 Modal

```html
<div class="modal" id="modal-schedule">
    <input type="hidden" id="schedule-id">
    <div>員工：<span id="schedule-name"></span></div>
    <div>日期：<span id="schedule-date"></span></div>
    <input type="text" id="schedule-in" placeholder="09:00">
    <input type="text" id="schedule-out" placeholder="18:00">
    <button id="btn-save">儲存</button>
    <button id="btn-delete">刪除</button>
</div>
```

點擊事件時填入資料，儲存送 PUT，刪除送 DELETE，完成後 `calendar.refetchEvents()`。

## 7. 驗證方式

```bash
# 確認路由
D:/Servers/PHP/php84/php.exe artisan route:list --name=schedule

# 瀏覽排班頁
http://127.0.0.1:8003/zh-hant/admin/hrm/schedule
```

測試步驟：
1. 左側拖放員工到某日 → 確認日格出現「姓名 09:00-18:00」
2. 點擊已排班次 → Modal 開啟，顯示正確時間
3. 修改時間後儲存 → 日格文字更新
4. 刪除班次 → 從日曆消失
5. 切換上週/下週 → 資料正確載入
