# 薪資項目

## 1. 設計目標

定義薪資項目的資料結構與計算邏輯，將出勤統計轉換為薪資明細。

### 公司綁定原則

所有薪資項目必須明確綁定至特定公司（`company_id` 為必填），不存在「系統通用」的薪資項目。每家公司各自擁有獨立的一套薪資項目，可自由調整金額、計算規則、新增或停用項目。

系統需支援三種場景：

- **全域項目**：同一公司內所有員工統一金額（如全勤獎金 2,000 元，全員適用）
- **個人項目**：依員工個別設定金額（如交通費，A 員工 2,000、B 員工 1,000）
- **自動計算項目**：從出勤資料自動計算（如遲到扣款、病假扣款）

### 1.1 資料流程

```
薪資項目定義（hrm_salary_items）── 每家公司各自定義項目
    +
員工薪資配置（hrm_employee_salary_items）── 指定個人金額 / 排除
    +
每月出勤統計（monthly_summaries, locked）── 提供出勤數據
    ↓ 月底批次計算（按公司篩選項目）
月薪資單（hrm_payrolls）── 1 筆/月/人
    +
薪資單明細（hrm_payroll_lines）── 每個項目一筆
```

### 1.2 與 ChatGPT 建議的差異

ChatGPT 建議 `pay_items → salary_structures → payroll_lines` 三層結構，本設計在此基礎上改進：

| 面向 | ChatGPT 建議 | 本設計 |
|------|-------------|--------|
| 項目適用範圍 | 未區分 | `scope` 三分法（global / individual / auto） |
| 自動計算 | 寫死在程式碼 | `calc_config` JSON 存規則，可調整不改程式 |
| 排除機制 | 未設計 | 員工配置表支援 `is_excluded`（排除特定人） |
| 歷史快照 | 無 | payroll_lines 保存 item_name / item_code 快照 |
| 時間區間 | 無 | effective_date / end_date 支援異動追溯 |


## 2. 資料表設計

### 2.1 hrm_salary_items（薪資項目定義）

```
hrm_salary_items
├── id
├── company_id              -- FK → companies, NOT NULL, CASCADE DELETE
│                              每家公司各自擁有獨立的薪資項目
├── code                    -- 項目代碼（varchar 30）；同 code 可存在於不同公司
├── name                    -- 項目名稱（varchar 100）
│
├── type                    -- earning / deduction（見 §3.1）
├── scope                   -- global / individual / auto（見 §3.2）
│
├── default_amount          -- 預設金額（decimal 10,0; nullable）
│                              global：全員統一金額
│                              individual：新增員工時的建議金額
│                              auto：不使用此欄位
│
├── calc_config             -- 自動計算設定（JSON, nullable; 見 §5）
│                              僅 scope = auto 時使用
│
├── is_taxable              -- 是否計入所得稅（boolean, default true）
├── is_insurable            -- 是否計入勞健保投保薪資（boolean, default true）
├── is_active               -- 是否啟用（boolean, default true）
├── sort_order              -- 排序（int, default 0）
├── note                    -- 備註（text, nullable）
│
├── timestamps
└── soft_deletes

UNIQUE(company_id, code)
INDEX(type)
INDEX(scope)
INDEX(is_active)
```

### 2.2 hrm_employee_salary_items（員工薪資配置）

```
hrm_employee_salary_items
├── id
├── employee_id             -- FK → employees, CASCADE
├── salary_item_id          -- FK → hrm_salary_items, CASCADE
│
├── amount                  -- 個人金額（decimal 10,0; nullable）
│                              individual 項目：必填，該員工的金額
│                              global 項目：nullable，填了代表覆寫預設金額
│
├── is_excluded             -- 是否排除（boolean, default false）
│                              僅 global 項目使用：true = 此人不適用該項目
│
├── effective_date          -- 生效日（date, nullable; null = 立即生效）
├── end_date                -- 結束日（date, nullable; null = 無期限）
│
├── note                    -- 備註（text, nullable; 如「搬家後調整交通費」）
├── timestamps
└── soft_deletes

UNIQUE(employee_id, salary_item_id, effective_date)
INDEX(employee_id)
INDEX(salary_item_id)
```

**用法說明**：

| scope | 有無配置記錄 | 行為 |
|-------|-------------|------|
| global | 無記錄 | 適用，金額 = default_amount |
| global | is_excluded = true | 不適用（如某人無全勤獎金資格） |
| global | amount = 3000 | 適用，金額覆寫為 3000 |
| individual | 無記錄 | 不適用 |
| individual | amount = 2000 | 適用，金額 = 2000 |
| auto | 無記錄 | 適用，金額由系統自動計算 |
| auto | is_excluded = true | 不適用 |

### 2.3 hrm_payrolls（月薪資單）

```
hrm_payrolls
├── id
├── employee_id             -- FK → employees, CASCADE
├── company_id              -- FK → companies（快照，計算當下的公司）
├── year_month              -- 年月 YYYY-MM（varchar 7）
├── monthly_summary_id      -- FK → monthly_summaries, nullable, SET NULL
│
├── base_salary             -- 底薪（decimal 10,0）
├── gross_pay               -- 應發合計（decimal 10,0）
├── total_deduction         -- 扣款合計（decimal 10,0; 正數）
├── net_pay                 -- 實領金額（decimal 10,0; = gross_pay - total_deduction）
│
├── status                  -- draft / confirmed / paid（見 §4）
├── note                    -- 備註（text, nullable）
├── confirmed_by            -- 確認人 ID（nullable, FK → users, nullOnDelete）
├── confirmed_at            -- 確認時間（nullable, timestamp）
├── paid_at                 -- 發薪時間（nullable, timestamp）
│
├── timestamps
└── soft_deletes

UNIQUE(employee_id, year_month)
INDEX(company_id, year_month)
INDEX(status)
```

### 2.4 hrm_payroll_lines（薪資單明細）

```
hrm_payroll_lines
├── id
├── payroll_id              -- FK → hrm_payrolls, CASCADE
├── salary_item_id          -- FK → hrm_salary_items, nullable, SET NULL
│
├── item_code               -- 快照：項目代碼（varchar 30）
├── item_name               -- 快照：項目名稱（varchar 100）
├── type                    -- 快照：earning / deduction
│
├── amount                  -- 金額（decimal 10,0; earning 正數, deduction 正數）
├── note                    -- 備註（text, nullable; 如「遲到 3 次共 45 分鐘」）
│
├── sort_order              -- 排序（int, default 0）
├── timestamps

INDEX(payroll_id)
INDEX(salary_item_id)
```

**快照欄位說明**：`item_code`、`item_name`、`type` 在建立明細時從 `hrm_salary_items` 複製。即使日後修改項目名稱，歷史薪資單不受影響。


## 3. 欄位說明

### 3.1 type（項目類型）

| 值 | 說明 | 範例 |
|----|------|------|
| `earning` | 收入項目 | 本薪、津貼、獎金、加班費 |
| `deduction` | 扣款項目 | 勞保、健保、遲到扣款、病假扣款 |

### 3.2 scope（適用範圍）

| 值 | 說明 | 金額來源 | 排除方式 | 範例 |
|----|------|----------|----------|------|
| `global` | 同公司全員統一金額 | `default_amount` | 員工配置 `is_excluded = true` | 全勤獎金、午餐補助 |
| `individual` | 按人設定金額 | 員工配置 `amount` | 不建立配置記錄 | 本薪、交通費、職務津貼 |
| `auto` | 從出勤資料自動計算 | `calc_config` 規則 | 員工配置 `is_excluded = true` | 遲到扣款、病假扣款、加班費 |

### 3.3 scope 選擇指引

```
這個項目同公司所有人金額都一樣嗎？
├── 是 → global（如全勤獎金 2000）
└── 否
    ├── 金額由出勤數據決定？
    │   ├── 是 → auto（如遲到扣款、加班費）
    │   └── 否 → individual（如交通費、本薪）
```


## 4. 狀態管理

### 4.1 薪資單狀態（hrm_payrolls.status）

| 狀態 | 說明 | 時機 |
|------|------|------|
| `draft` | 草稿 | 系統計算產生，尚未確認 |
| `confirmed` | 已確認 | HR 確認薪資正確 |
| `paid` | 已發薪 | 實際撥款完成 |

**狀態流轉**：

```
（月出勤統計 approved）
    ↓ 觸發薪資計算
draft（自動產生薪資單 + 明細）
    ↓ HR 逐一或批次確認
confirmed（鎖定月出勤統計為 locked）
    ↓ 發薪完成
paid（記錄 paid_at）
```

### 4.2 與月出勤統計的連動

| 事件 | 月出勤統計狀態 | 薪資單狀態 |
|------|---------------|-----------|
| 出勤統計審核通過 | approved | — |
| 觸發薪資計算 | approved | draft（產生） |
| HR 確認薪資 | locked | confirmed |
| 需重新計算 | approved（解鎖） | draft（重新產生） |


## 5. 自動計算邏輯（calc_config）

`scope = auto` 的項目透過 `calc_config` JSON 定義計算規則，系統從 `monthly_summaries` 讀取數據自動算出金額。

### 5.1 全勤獎金

條件式：所有條件皆滿足時才發放。

```json
{
  "method": "conditional",
  "conditions": [
    { "field": "late_count", "op": "=", "value": 0 },
    { "field": "early_leave_count", "op": "=", "value": 0 },
    { "field": "absent_days", "op": "=", "value": 0 },
    { "field": "personal_leave_days", "op": "=", "value": 0 }
  ],
  "amount_source": "default_amount"
}
```

**說明**：當月無遲到、無早退、無缺勤、無事假 → 發放 `default_amount`（如 2,000 元）。全勤獎金雖為 auto 計算，但金額固定，取 `default_amount`。

> 注意：病假與特休不影響全勤。依勞基法，勞工請特休、病假（30 日內）不應扣全勤。

### 5.2 遲到扣款

> **計算規則尚未確定**，以下列出三種方案，實作時擇一或組合使用。

**方案 A：按分鐘扣款**

```json
{
  "method": "per_unit",
  "source_field": "late_minutes",
  "rate": 10,
  "note_template": "遲到 {late_count} 次，共 {late_minutes} 分鐘"
}
```

範例：遲到 45 分鐘 → 扣 450 元

**方案 B：按次扣款**

```json
{
  "method": "per_unit",
  "source_field": "late_count",
  "rate": 200,
  "note_template": "遲到 {late_count} 次"
}
```

範例：遲到 3 次 → 扣 600 元

**方案 C：階梯式扣款**

```json
{
  "method": "tiered",
  "source_field": "late_minutes",
  "tiers": [
    { "min": 1,  "max": 10,  "amount": 100 },
    { "min": 11, "max": 30,  "amount": 300 },
    { "min": 31, "max": null, "amount": "half_day_salary" }
  ],
  "apply_per": "occurrence",
  "note_template": "遲到 {late_count} 次"
}
```

說明：每次遲到依分鐘數落入不同級距，各次加總。

### 5.3 病假扣款

依勞基法，年度前 30 天病假扣半薪。

```json
{
  "method": "daily_rate",
  "source_field": "sick_leave_days",
  "ratio": 0.5,
  "note_template": "病假 {sick_leave_days} 天，扣半薪"
}
```

計算公式：`sick_leave_days × (base_salary ÷ 30) × 0.5`

> 超過年度 30 天的病假處理方式待定。

### 5.4 事假扣款

事假不給薪，全額扣除。

```json
{
  "method": "daily_rate",
  "source_field": "personal_leave_days",
  "ratio": 1.0,
  "note_template": "事假 {personal_leave_days} 天"
}
```

計算公式：`personal_leave_days × (base_salary ÷ 30)`

### 5.5 加班費

依 [0301 行事曆管理](0301_行事曆管理.md) 定義的加班費率。

```json
{
  "method": "overtime",
  "rules": [
    {
      "source_field": "weekday_overtime_minutes",
      "label": "平日加班",
      "tiers": [
        { "hours": 2, "rate": 1.34 },
        { "hours": null, "rate": 1.67 }
      ]
    },
    {
      "source_field": "holiday_overtime_minutes",
      "label": "假日加班",
      "rate": 2.0
    }
  ],
  "base": "hourly_rate",
  "note_template": "平日加班 {weekday_overtime_minutes} 分鐘、假日加班 {holiday_overtime_minutes} 分鐘"
}
```

時薪基準：`base_salary ÷ 30 ÷ 8`（月薪制，每月 30 日、每日 8 小時）

> 注意：休息日加班（rest_day）的費率較複雜（前 2h ×1⅓、3-8h ×1⅔、9-12h ×2⅔），需在 daily_attendances 中區分 day_type，分別計算後加總至 monthly_summaries。目前 monthly_summaries 僅區分 weekday / holiday，若需更精細可擴充欄位。


## 6. 程式架構（Strategy Pattern）

### 6.1 目錄結構

```
app/Portals/Ocadmin/Modules/Hrm/Salary/
├── SalaryItemController.php                 -- 薪資項目 CRUD（按公司篩選）
├── PayrollController.php                    -- 薪資單管理
└── Services/
    ├── SalaryCalculatorService.php          -- 主服務：按公司篩選項目、分派計算
    ├── CalculationResult.php               -- 計算結果 DTO
    ├── Contracts/
    │   └── ItemCalculatorInterface.php      -- Calculator 介面
    └── Calculators/
        ├── ConditionalCalculator.php        -- method=conditional（全勤獎金）
        ├── PerUnitCalculator.php            -- method=per_unit（按次/按分鐘）
        ├── DailyRateCalculator.php          -- method=daily_rate（病假/事假/缺勤）
        ├── LatePenaltyCalculator.php        -- method=late_penalty（遲到，含寬限）
        ├── OvertimeCalculator.php           -- method=overtime（加班費）
        └── InsuranceCalculator.php          -- method=insurance（勞健保費）
```

### 6.2 核心介面

```php
// Services/Contracts/ItemCalculatorInterface.php

interface ItemCalculatorInterface
{
    /**
     * @param  object         $calcConfig  來自 salary_items.calc_config 的 JSON
     * @param  Employee       $employee
     * @param  MonthlySummary $summary
     * @return CalculationResult  包含 amount（int）和 note（string）
     */
    public function calculate(
        object $calcConfig,
        Employee $employee,
        MonthlySummary $summary
    ): CalculationResult;
}
```

### 6.3 主服務：分派邏輯

```php
// Services/SalaryCalculatorService.php

class SalaryCalculatorService
{
    /**
     * calc_config.method → Calculator class 對照表
     * 新增計算方式時，只需：1.建立 Calculator class  2.加入此陣列
     */
    protected array $calculators = [
        'conditional'  => ConditionalCalculator::class,
        'per_unit'     => PerUnitCalculator::class,
        'daily_rate'   => DailyRateCalculator::class,
        'late_penalty' => LatePenaltyCalculator::class,
        'overtime'     => OvertimeCalculator::class,
        'insurance'    => InsuranceCalculator::class,
    ];

    /**
     * 計算單一 auto 項目的金額
     * 各公司透過不同的 calc_config JSON 來驅動不同的計算行為，
     * Calculator class 本身是全域共用的。
     */
    public function calculateAutoItem(SalaryItem $item, Employee $employee, MonthlySummary $summary): CalculationResult
    {
        $calcConfig = json_decode($item->calc_config);
        $calculatorClass = $this->calculators[$calcConfig->method]
            ?? throw new \InvalidArgumentException("未知的計算方式: {$calcConfig->method}");

        return app($calculatorClass)->calculate($calcConfig, $employee, $summary);
    }
}
```

### 6.4 Calculator 範例：遲到扣款（含寬限）

處理規則如「每月遲到 15 分鐘以內，前 3 次不扣，之後按分鐘扣」：

```php
// Services/Calculators/LatePenaltyCalculator.php

class LatePenaltyCalculator implements ItemCalculatorInterface
{
    public function calculate(object $calcConfig, Employee $employee, MonthlySummary $summary): CalculationResult
    {
        // 需要逐日資料（monthly_summaries 只有合計，無法判斷寬限）
        [$startDate, $endDate] = yearMonthRange($summary->year_month);

        $dailyLates = DailyAttendance::where('employee_id', $employee->id)
            ->whereBetween('work_date', [$startDate, $endDate])
            ->where('is_late', true)
            ->orderBy('work_date')
            ->get();

        $deduction = 0;
        $graceUsed = 0;
        $penaltyCount = 0;
        $penaltyMinutes = 0;

        foreach ($dailyLates as $record) {
            // 在寬限範圍內（如 ≤ 15 分鐘）且寬限次數未用完
            if ($record->late_minutes <= ($calcConfig->grace_threshold_minutes ?? 0)
                && $graceUsed < ($calcConfig->grace_count ?? 0)
            ) {
                $graceUsed++;
                continue;
            }

            // 超過寬限 → 扣款
            $deduction += $record->late_minutes * $calcConfig->rate_per_minute;
            $penaltyCount++;
            $penaltyMinutes += $record->late_minutes;
        }

        return new CalculationResult(
            amount: $deduction,
            note: "遲到 {$dailyLates->count()} 次，寬限 {$graceUsed} 次，扣款 {$penaltyCount} 次共 {$penaltyMinutes} 分鐘"
        );
    }
}
```

對應的 `calc_config`：

```json
{
  "method": "late_penalty",
  "grace_threshold_minutes": 15,
  "grace_count": 3,
  "rate_per_minute": 10
}
```

### 6.5 規則變化時改哪裡？

| 變化類型 | 改什麼 | 誰來改 | 範例 |
|---------|--------|--------|------|
| 調整參數 | 該公司的 `calc_config` JSON | HR（管理介面） | A 公司寬限從 3 次改 5 次 |
| 切換計算方式 | 該公司的 `calc_config.method` | HR（管理介面） | B 公司從 `late_penalty` 換成 `per_unit` |
| 公司間差異 | 各公司各自調整 `calc_config` / `default_amount` | HR（管理介面） | A 公司全勤 2000、B 公司全勤 3000 |
| 全新計算邏輯 | 新增 Calculator class + 註冊 | 工程師 | 累進罰則、分級扣款 |

**設計原則**：
- 參數變 → 改 JSON（不動程式）
- 邏輯變 → 改/加 Calculator class
- 公司差異 → 各公司有各自的 salary_items 記錄，Calculator class 全域共用

### 6.6 多公司差異範例

同一個 `code`（如 `full_attendance_bonus`）在不同公司可以有完全不同的設定：

| 公司 | default_amount | calc_config 差異 | 說明 |
|------|---------------|-----------------|------|
| A 公司 | 2,000 | 條件：無遲到、無缺勤、無事假 | 標準設定 |
| B 公司 | 3,000 | 條件：僅看缺勤天數 | B 公司全勤條件較寬鬆 |
| C 公司 | （不建立此項目） | — | C 公司沒有全勤獎金 |

其他常見差異：

| 場景 | 做法 |
|------|------|
| A 公司遲到寬限 3 次、B 公司寬限 5 次 | 各自的 `late_deduction` 項目 `calc_config.grace_count` 不同 |
| B 公司不發午餐補助 | B 公司不建立 `lunch_allowance`，或建立後設 `is_active = false` |
| C 公司有獨有的「績效獎金」 | 只在 C 公司建立 `performance_bonus` 項目 |


## 7. 預設薪資項目（Seeder）

Seeder 會為每家啟用中的公司各建立一套完整的預設薪資項目。各公司在建立後可自行調整金額、計算規則、新增或停用項目。

公開方法 `seedForCompany(int $companyId)` 可供新增公司時單獨呼叫。

### 7.1 收入項目（earning）

| code | name | scope | default_amount | 說明 |
|------|------|-------|---------------|------|
| `base_salary` | 本薪 | individual | — | 每位員工個別設定 |
| `duty_allowance` | 職務津貼 | individual | — | 依職務等級設定 |
| `supervisor_allowance` | 主管津貼 | individual | — | 僅主管適用 |
| `skill_allowance` | 技術津貼 | individual | — | 依技能認證設定 |
| `transport_allowance` | 交通津貼 | individual | — | 依通勤距離設定 |
| `lunch_allowance` | 午餐補助 | global | 自訂 | 全員統一金額 |
| `phone_allowance` | 電話補貼 | individual | — | 依職務需求設定 |
| `full_attendance_bonus` | 全勤獎金 | auto | 自訂 | 條件：無遲到、無缺勤、無事假 |
| `overtime_pay` | 加班費 | auto | — | 依 0301 費率自動計算 |

### 7.2 扣款項目（deduction）

| code | name | scope | default_amount | 說明 |
|------|------|-------|---------------|------|
| `labor_insurance` | 勞保費 | auto | — | 依投保級距自動計算（見 [0309](0309_勞健保.md)） |
| `health_insurance` | 健保費 | auto | — | 依投保級距自動計算（見 [0309](0309_勞健保.md)） |
| `late_deduction` | 遲到扣款 | auto | — | 規則待定（見 §5.2） |
| `early_leave_deduction` | 早退扣款 | auto | — | 比照遲到扣款規則 |
| `sick_leave_deduction` | 病假扣款 | auto | — | 半薪（見 §5.3） |
| `personal_leave_deduction` | 事假扣款 | auto | — | 全額（見 §5.4） |
| `absent_deduction` | 缺勤扣款 | auto | — | 全額（無請假之缺勤） |

### 7.3 全勤獎金的特殊說明

全勤獎金的 `scope` 設為 `auto` 而非 `global`，因為它有條件判斷（出勤完美才發放）。
金額來自 `default_amount`，適用判斷來自 `calc_config` 的 conditions。

若某員工不適用全勤獎金制度（如高階主管），在 `hrm_employee_salary_items` 建立一筆 `is_excluded = true` 的記錄。


## 8. 薪資計算流程

### 8.1 月薪計算步驟

```
1. 確認月出勤統計（monthly_summaries）狀態為 approved
2. 取得該員工所屬公司的所有適用薪資項目（按 company_id 篩選）：
   a. scope = global：取 default_amount，排除 is_excluded 的員工
   b. scope = individual：取員工配置的 amount
   c. scope = auto：依 calc_config 從月統計資料計算
3. 產生 hrm_payrolls 記錄（status = draft）
4. 逐項產生 hrm_payroll_lines 記錄（含快照欄位）
5. 計算 gross_pay（earning 合計）、total_deduction（deduction 合計）、net_pay
```

### 8.2 適用項目判斷邏輯

```
function getApplicableItems(Employee $employee, string $yearMonth): Collection
{
    // 只取該員工所屬公司的薪資項目
    $items = SalaryItem::where('company_id', $employee->company_id)
        ->where('is_active', true)
        ->orderBy('sort_order')
        ->get();

    $configs = EmployeeSalaryItem::where('employee_id', $employee->id)
        ->whereDate('effective_date', '<=', $yearMonth末日)
        ->where(fn($q) => $q->whereNull('end_date')->orWhereDate('end_date', '>=', $yearMonth首日))
        ->get()
        ->keyBy('salary_item_id');

    return $items->filter(function ($item) use ($configs) {
        $config = $configs->get($item->id);

        if ($item->scope === 'global' || $item->scope === 'auto') {
            // 有配置且被排除 → 不適用
            return !($config && $config->is_excluded);
        }

        if ($item->scope === 'individual') {
            // 必須有配置記錄且未被排除
            return $config && !$config->is_excluded;
        }
    });
}
```

### 8.3 金額計算邏輯

```
function calculateAmount(SalaryItem $item, ?EmployeeSalaryItem $config, MonthlySummary $summary): int
{
    switch ($item->scope) {
        case 'global':
            return $config?->amount ?? $item->default_amount;

        case 'individual':
            return $config->amount;

        case 'auto':
            return $this->calculateAuto($item, $summary);
    }
}
```

## 9. Employee 表擴充

員工表需新增底薪欄位，供薪資計算使用：

```diff
employees
+ base_salary             -- 月底薪（decimal 10,0; nullable）
+ salary_type             -- 計薪方式：monthly / hourly（varchar 10, default 'monthly'）
```

`base_salary` 同時記錄在 `hrm_employee_salary_items`（code = `base_salary`），兩者需同步。
建議以 `hrm_employee_salary_items` 為主要資料來源，`employees.base_salary` 為方便查詢的冗餘欄位。
