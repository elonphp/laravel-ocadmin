# 多層選項連動

---

## 一、概述

本系統的「多層選項連動」功能允許後台管理員定義選項之間的層級連動關係，使前端使用者在選擇上一層選項後，下一層選項自動篩選為對應的可用項目。

### 範例情境：汽車規格選配

```
廠牌 → 車型 → 車款
```

- 選擇「Toyota」後，車型只顯示 Altis、Yaris
- 選擇「Honda」後，車型只顯示 Civic、Fit
- 選擇「Toyota → Altis」後，車款只顯示旗艦型、豪華型

### 驅動模式

| 模式 | 說明 | 適用場景 |
|------|------|----------|
| **資料型驅動** | 從現有資料反查可用選項 | 二手車鑑價（只顯示有車可賣的選項） |
| **結構型驅動** | 預先定義選項之間的關聯 | 產品規格選配（所有可能的組合都要顯示） |

**本專案採用結構型驅動**——選項關聯由後台管理員預先設定，與實際庫存無關。

## 二、資料表設計

所有資料表使用 `clg_` 前綴（Catalog 模組）。

### 2.1 資料表總覽

| 資料表 | 說明 | Migration |
|--------|------|-----------|
| `clg_options` | 選項定義 | `2026_02_10_000001` |
| `clg_option_translations` | 選項多語系 | 同上 |
| `clg_option_values` | 選項值 | 同上 |
| `clg_option_value_translations` | 選項值多語系 | 同上 |
| `clg_option_value_groups` | 連動群組 | `2026_02_16_000001` |
| `clg_option_value_group_translations` | 連動群組多語系 | 同上 |
| `clg_option_value_group_levels` | 連動群組層級 | 同上 |
| `clg_option_value_links` | 選項值連動關係 | 同上 |

### 2.2 clg_options（選項定義）

定義選項的「類型」，如：廠牌、車型、車款。

| 欄位 | 類型 | 說明 |
|------|------|------|
| `id` | bigint | PK |
| `code` | varchar(50) | 識別碼，唯一（如 `brand`、`model`、`trim`） |
| `type` | varchar(20) | 輸入類型，預設 `select` |
| `sort_order` | int | 排序，預設 0 |
| `created_at` | timestamp | |
| `updated_at` | timestamp | |

可用類型：`select`、`radio`、`checkbox`、`text`、`textarea`、`file`、`date`、`time`、`datetime`

**範例資料：**

| id | code | type | sort_order |
|----|------|------|------------|
| 1 | brand | select | 0 |
| 2 | model | select | 1 |
| 3 | trim | select | 2 |

### 2.3 clg_option_translations（選項多語系）

| 欄位 | 類型 | 說明 |
|------|------|------|
| `id` | bigint | PK |
| `option_id` | bigint | FK → clg_options |
| `locale` | varchar(10) | 語系代碼 |
| `name` | varchar(128) | 顯示名稱 |

**唯一約束**：`UNIQUE (option_id, locale)`

**範例資料：**

| option_id | locale | name |
|-----------|--------|------|
| 1 | zh_Hant | 廠牌 |
| 1 | en | Brand |
| 2 | zh_Hant | 車型 |
| 2 | en | Model |
| 3 | zh_Hant | 車款 |
| 3 | en | Trim |

### 2.4 clg_option_values（選項值）

| 欄位 | 類型 | 說明 |
|------|------|------|
| `id` | bigint | PK |
| `option_id` | bigint | FK → clg_options |
| `code` | varchar(50) | 識別碼 |
| `image` | varchar(255) | 選項圖片（如廠牌 Logo） |
| `sort_order` | int | 排序，預設 0 |
| `created_at` | timestamp | |
| `updated_at` | timestamp | |

**範例資料：**

| id | option_id | code | sort_order | 說明 |
|----|-----------|------|------------|------|
| 1 | 1 | toyota | 0 | 廠牌：Toyota |
| 2 | 1 | honda | 1 | 廠牌：Honda |
| 3 | 1 | ford | 2 | 廠牌：Ford |
| 4 | 2 | altis | 0 | 車型：Altis |
| 5 | 2 | yaris | 1 | 車型：Yaris |
| 6 | 2 | civic | 2 | 車型：Civic |
| 7 | 2 | fit | 3 | 車型：Fit |
| 8 | 2 | focus | 4 | 車型：Focus |
| 9 | 2 | kuga | 5 | 車型：Kuga |
| 10 | 3 | flagship | 0 | 車款：旗艦型 |
| 11 | 3 | luxury | 1 | 車款：豪華型 |
| 12 | 3 | classic | 2 | 車款：經典型 |

### 2.5 clg_option_value_groups（連動群組）

定義一組連動選項的集合，例如「汽車規格」群組包含廠牌→車型→車款三層。

| 欄位 | 類型 | 說明 |
|------|------|------|
| `id` | bigint | PK |
| `code` | varchar(50) | 唯一識別碼（如 `car_spec`） |
| `sort_order` | int | 排序，預設 0 |
| `is_active` | boolean | 啟用狀態，預設 true |
| `created_at` | timestamp | |
| `updated_at` | timestamp | |

### 2.6 clg_option_value_group_levels（連動群組層級）

定義群組內各選項的連動順序。

| 欄位 | 類型 | 說明 |
|------|------|------|
| `id` | bigint | PK |
| `option_value_group_id` | bigint | FK → clg_option_value_groups |
| `option_id` | bigint | FK → clg_options |
| `level` | tinyint | 層級（0, 1, 2...） |

**唯一約束**：
- `UNIQUE (option_value_group_id, option_id)` — 同群組不可重複選項
- `UNIQUE (option_value_group_id, level)` — 同群組每層只有一個選項

**範例資料（汽車規格群組）：**

| option_value_group_id | option_id | level |
|-----------------------|-----------|-------|
| 1 | 1 (廠牌) | 0 |
| 1 | 2 (車型) | 1 |
| 1 | 3 (車款) | 2 |

### 2.7 clg_option_value_links（選項值連動關係）

**核心表**——定義「父選項值」與「子選項值」的連動關係。

| 欄位 | 類型 | 說明 |
|------|------|------|
| `id` | bigint | PK |
| `parent_option_value_id` | bigint | FK → clg_option_values |
| `child_option_value_id` | bigint | FK → clg_option_values |

**唯一約束**：`UNIQUE (parent_option_value_id, child_option_value_id)`

**範例資料：**

選擇「Toyota」後，可選車型為 Altis、Yaris：

| parent_option_value_id | child_option_value_id |
|------------------------|-----------------------|
| 1 (Toyota) | 4 (Altis) |
| 1 (Toyota) | 5 (Yaris) |
| 2 (Honda) | 6 (Civic) |
| 2 (Honda) | 7 (Fit) |
| 3 (Ford) | 8 (Focus) |
| 3 (Ford) | 9 (Kuga) |

選擇「Toyota → Altis」後，可選車款：

| parent_option_value_id | child_option_value_id |
|------------------------|-----------------------|
| 4 (Altis) | 10 (旗艦型) |
| 4 (Altis) | 11 (豪華型) |
| 5 (Yaris) | 11 (豪華型) |
| 5 (Yaris) | 12 (經典型) |

## 四、Model 與關聯

### 4.1 Option

```php
// app/Models/Catalog/Option.php

class Option extends Model
{
    use HasTranslation;

    protected $table = 'clg_options';
    protected $fillable = ['code', 'type', 'sort_order'];
    public array $translatedAttributes = ['name'];

    public const CHOICE_TYPES = ['select', 'radio', 'checkbox'];

    public function optionValues(): HasMany
    {
        return $this->hasMany(OptionValue::class)->orderBy('sort_order');
    }

    public function valueGroups(): BelongsToMany
    {
        return $this->belongsToMany(
            OptionValueGroup::class,
            'clg_option_value_group_levels',
            'option_id',
            'option_value_group_id'
        )->withPivot('level')->orderByPivot('level');
    }
}
```

```php
class OptionValue extends Model

    protected $table = 'clg_option_values';

    public array $translatedAttributes = ['name'];

    public function option(): BelongsTo { ... }

    // 此值作為「父」→ 查詢連動的子值
```

## 五、後台管理流程

後台路徑：`/{locale}/admin/catalog/`

### 5.1 步驟一：建立選項（Option）

路徑：`option/create`

建立三個選項，各自包含多個選項值：

| 選項 | code | 類型 | 選項值 |
|------|------|------|--------|
| 廠牌 | brand | select | Toyota, Honda, Ford |
| 車型 | model | select | Altis, Yaris, Civic, Fit, Focus, Kuga |
| 車款 | trim | select | 旗艦型, 豪華型, 經典型 |

### 5.2 步驟二：建立連動群組（OptionValueGroup）

路徑：`option-value-group/create`

建立群組「汽車規格」（code: `car_spec`），設定三個層級：

| 層級 | 選項 |
|------|------|
| Level 0 | 廠牌 (brand) |
| Level 1 | 車型 (model) |
| Level 2 | 車款 (trim) |

### 5.3 步驟三：設定連動關係（OptionValueLink）

路徑：`option-value-link?group_id=1`

選擇群組後，頁面分為**連動設定**與**連動測試**兩區：

```
┌─ 連動設定 ──────────────────┐  ┌─ 連動測試 ──────────────┐
│                              │  │                          │
│ [Level 0→1] [Level 1→2]     │  │ 廠牌：[Toyota ▼]         │
│                              │  │ 車型：[Altis  ▼]  ← 連動 │
│ 父項值：                     │  │ 車款：[旗艦型 ▼]  ← 連動 │
│ ● Toyota                    │  │                          │
│   ☑ Altis  ☑ Yaris          │  │                          │
│ ○ Honda                     │  │                          │
│ ○ Ford                      │  │                          │
│          [儲存]              │  │                          │
└──────────────────────────────┘  └──────────────────────────┘
```

操作方式：
1. 選擇 Level 0→1 頁籤
2. 點擊父值「Toyota」，勾選子值「Altis、Yaris」，儲存
3. 點擊父值「Honda」，勾選子值「Civic、Fit」，儲存
4. 切換 Level 1→2 頁籤，重複設定車型→車款的連動
5. 右側「連動測試」可即時驗證設定結果

## 七、前台 / API 應用

目前連動端點位於後台路由，以下說明如何將連動功能暴露給前台或外部 API 使用。

### 7.1 建議 API 路由設計

```php
// routes/api.php

Route::prefix('v1/catalog')->group(function () {
    // 取得連動群組結構（含所有層級與選項值）
    Route::get('/cascade-groups/{code}', [CascadeController::class, 'group']);

    // 根據父選項值取得可選的子選項值
    Route::get('/cascade/{parentValueId}/children', [CascadeController::class, 'children']);
});
```

### 7.2 API Controller 範例

```php
namespace App\Http\Controllers\Api\V1;

use App\Models\Catalog\OptionValueGroup;
use App\Models\Catalog\OptionValueLink;

class CascadeController extends Controller
{
    /**
     * GET /api/v1/catalog/cascade-groups/car_spec
     * 取得群組完整結構，前端初始化連動選單用
     */
    public function group(string $code)
    {
        $group = OptionValueGroup::where('code', $code)
            ->where('is_active', true)
            ->with([
                'translations',
                'levels.option.translations',
                'levels.option.optionValues.translations',
            ])
            ->firstOrFail();

        return response()->json([
            'data' => [
                'id'     => $group->id,
                'name'   => $group->name,
                'code'   => $group->code,
                'levels' => $group->levels->map(fn ($level) => [
                    'level'       => $level->level,
                    'option_code' => $level->option->code,
                    'option_name' => $level->option->name,
                    'values'      => $level->option->optionValues->map(fn ($v) => [
                        'id'   => $v->id,
                        'code' => $v->code,
                        'name' => $v->name,
                    ]),
                ]),
            ],
        ]);
    }

    /**
     * GET /api/v1/catalog/cascade/1/children
     * 根據已選父值，回傳可選的子選項值
     */
    public function children(int $parentValueId)
    {
        $children = OptionValueLink::where('parent_option_value_id', $parentValueId)
            ->with('childValue.translations')
            ->get()
            ->map(fn ($link) => [
                'id'        => $link->childValue->id,
                'code'      => $link->childValue->code,
                'name'      => $link->childValue->name,
                'option_id' => $link->childValue->option_id,
            ]);

        return response()->json(['data' => $children]);
    }
}
```

### 7.3 前端連動流程

```
1. 頁面載入
   → GET /api/v1/catalog/cascade-groups/car_spec
   → 取得群組結構，渲染 Level 0（廠牌）下拉選單

2. 使用者選擇廠牌 = Toyota (id=1)
   → GET /api/v1/catalog/cascade/1/children
   → 回傳 [{id:4, name:"Altis"}, {id:5, name:"Yaris"}]
   → 渲染 Level 1（車型）下拉選單

3. 使用者選擇車型 = Altis (id=4)
   → GET /api/v1/catalog/cascade/4/children
   → 回傳 [{id:10, name:"旗艦型"}, {id:11, name:"豪華型"}]
   → 渲染 Level 2（車款）下拉選單

4. 使用者改選廠牌 = Honda (id=2)
   → 清空 Level 1、Level 2
   → GET /api/v1/catalog/cascade/2/children
   → 重新渲染 Level 1
```

### 7.4 前端實作範例（JavaScript）

```javascript
async function loadChildren(parentValueId, nextSelectEl) {
    const res = await fetch(`/api/v1/catalog/cascade/${parentValueId}/children`);
    const { data } = await res.json();

    nextSelectEl.innerHTML = '<option value="">-- 請選擇 --</option>';
    data.forEach(item => {
        nextSelectEl.innerHTML += `<option value="${item.id}">${item.name}</option>`;
    });
    nextSelectEl.disabled = false;
}

// 綁定每層 select 的 change 事件
document.querySelectorAll('.cascade-select').forEach((select, index, selects) => {
    select.addEventListener('change', function () {
        // 清空後續所有層級
        for (let i = index + 1; i < selects.length; i++) {
            selects[i].innerHTML = '<option value="">-- 請選擇 --</option>';
            selects[i].disabled = true;
        }

        if (this.value && index + 1 < selects.length) {
            loadChildren(this.value, selects[index + 1]);
        }
    });
});
```

### 7.5 多層連動一次查詢（進階）

若前端需要一次取得多層已選值的完整路徑驗證：

```php
// 驗證選擇鏈是否合法：Toyota → Altis → 旗艦型
$selectedIds = [1, 4, 10]; // [廠牌, 車型, 車款]

for ($i = 0; $i < count($selectedIds) - 1; $i++) {
    $exists = OptionValueLink::where('parent_option_value_id', $selectedIds[$i])
        ->where('child_option_value_id', $selectedIds[$i + 1])
        ->exists();

    if (!$exists) {
        return response()->json(['error' => '選項組合不合法'], 422);
```
