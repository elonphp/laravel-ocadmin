# 系統架構

## 核心觀點：這是一個 Laravel 專案

本系統的基礎是 **Laravel**。所有的路由、認證、授權、資料庫、佇列、快取 — 全部由 Laravel 掌控。

在 Laravel 之上，採用 **Portal 多入口架構**，讓同一個 Laravel 專案同時服務不同對象（後台管理員、員工、門市人員、前台訪客），每個入口有獨立的技術棧、角色與 URL 前綴。

```
┌──────────────────────────────────────────────────────────────┐
│                         Laravel 12                           │
│              路由 / 認證 / 授權 / ORM / 佇列 / 快取            │
│                                                              │
│   ┌──────────────┐  ┌──────────────┐  ┌──────────────┐      │
│   │   Ocadmin    │  │     ESS      │  │  POS Portal  │      │
│   │   Portal     │  │   Portal     │  │  （未來）     │      │
│   │              │  │              │  │              │      │
│   │ Blade+jQuery │  │ Inertia+React│  │ Blade+       │      │
│   │ Bootstrap 5  │  │ Tailwind CSS │  │ Tailwind CSS │      │
│   └──────────────┘  └──────────────┘  └──────────────┘      │
│                                                              │
│   ┌─────────────────────────────────────────────────────┐   │
│   │           共用：Models / Migrations / Services       │   │
│   └─────────────────────────────────────────────────────┘   │
└──────────────────────────────────────────────────────────────┘
```

Portal 是本系統最重要的設計決策，詳見 [0105_Portal概述.md](0105_Portal概述.md)。

## 技術棧

| 層 | 技術 | 說明 |
|----|------|------|
| **基礎框架** | Laravel 12 | 路由、認證、授權、ORM、佇列 |
| **語言** | PHP 8.4 | 後端語言 |
| **資料庫** | MySQL / MariaDB | 資料持久化 |
| **Ocadmin Portal** | Blade + jQuery + Bootstrap 5 | 後台管理介面 |
| **ESS Portal**（未實作） | Inertia.js + React 19 + Tailwind CSS 4 | 員工自助介面 |
| **POS Portal**（未來） | Blade + Tailwind CSS 4 + DaisyUI | 門市收銀介面 |
| **建構工具** | Vite 7 | 前端資源打包 |

各 Portal 的前端技術棧完全獨立，互不干擾。選型依使用者情境決定，詳見 [0500_Inertia架構.md](0500_Inertia架構.md)。

## 為什麼不採用前後端完全分離

本系統的使用情境：

| 問題 | 答案 | 結論 |
|------|------|------|
| 需要 Mobile App 共用 API？ | 否 | 不需要獨立 API |
| 需要第三方系統串接？ | 否 | 不需要 API-first |
| 有獨立前端團隊？ | 否 | 不需要分離開發 |
| 面向大眾、需要 SEO？ | 官網需要，後台不需要 | 官網 Portal 另行評估 |

**採用 Portal 架構的優勢：** 各 Portal 各自選擇最適合的技術棧（Blade、Inertia、純 API 皆可），而不是強制整個系統使用同一種前端方式。

需要 API 時（如行動 App 或第三方串接），可直接在對應 Portal 加入 API 路由，與現有 Portal 並存，不需重構整個系統。

## 業務模組範例

| 業務功能 | Portal | Controller | 路由前綴 |
|----------|--------|-----------|---------|
| 系統設定 | Ocadmin | `Ocadmin\System\SettingController` | `/admin/system/setting` |
| 員工人資管理 | Ocadmin | `Ocadmin\Hrm\EmployeeController` | `/admin/hrm/employees` |
| 員工個人資料（ESS） | ESS | `ESS\Hrm\Employee\ProfileController` | `/ess/profile` |

完整的 Ocadmin 模組規範詳見 [0106_Ocadmin程式規範.md](0106_Ocadmin程式規範.md)。
