# 權限機制

> 建立日期：2026-02

---

## 一、概述

### 1.1 技術選型

採用 [Spatie Laravel Permission](https://spatie.be/docs/laravel-permission) 作為權限管理基礎。

| 項目 | 說明 |
|------|------|
| 套件 | spatie/laravel-permission ^6.24 |
| Guard | 單一 Guard (`web`) |
| 角色命名 | Portal 前綴：`{portal}.{role_name}`（`super_admin` 例外） |
| 權限命名 | 四段式：`{portal}.{module}.{resource}.{action}` |
| 萬用權限 | Spatie Wildcard Permission（`enable_wildcard_permission => true`） |
| 多語支援 | 角色與權限的 `display_name`、`note` 透過翻譯表實現 |
| 資料表前綴 | `acl_`（透過 `config/permission.php` 設定） |

### 1.2 權限類型

| 類型 | 說明 | 實現方式 |
|------|------|----------|
| **功能權限** | 能否執行某操作 | Spatie Permission |
| **資料權限** | 能看到哪些資料 | 組織層級 + Policy |

### 1.3 現有基礎設施

- [x] `spatie/laravel-permission` 套件安裝
- [x] `config/permission.php` 設定（`acl_` 前綴、wildcard 啟用）
- [x] ACL 資料庫遷移（含翻譯表）
- [x] 自訂 Model：`App\Models\Acl\Role`、`App\Models\Acl\Permission`（含多語翻譯）
- [x] `User` Model 已掛載 `HasRoles` trait
- [x] 角色 Seeder (`AclRoleSeeder`)
- [ ] 權限 Seeder
- [ ] 路由級權限檢查

## 二、角色設計

### 2.1 命名規範

**格式：`{portal}.{role_name}`**

```
hrm.hr_manager
 │      │
 │      └── 角色名稱（snake_case）
 └── Portal 前綴
```

例外：`super_admin` 為全域角色，不帶 Portal 前綴。

> Portal 的定義與架構決策請見 [0105_Portal概述.md](0105_Portal概述.md)。

### 2.2 角色清單

#### 全域角色

| 角色 name | 中文名稱 | 說明 |
|-----------|----------|------|
| `super_admin` | 超級管理員 | 最高權限，僅用於 Ocadmin |

#### HRM Portal 角色（hrm.*）

**HR 管理角色：**

| 角色 name | 中文名稱 | 說明 |
|-----------|----------|------|
| `hrm.hr_manager` | HR 主管 | 管理所有 HR 功能，含薪資、報表 |
| `hrm.hr_operator` | HR 管理員 | 日常 HR 操作：員工資料、出勤、請假 |
| `hrm.payroll_manager` | 薪資主管 | 薪資計算、薪資單管理 |

**員工自助服務角色：**

| 角色 name | 中文名稱 | 說明 |
|-----------|----------|------|
| `hrm.dept_manager` | 部門主管 | 審核下屬請假/出勤，檢視部門資料 |
| `hrm.employee` | 一般員工 | 查看個人資料、打卡、申請請假 |

#### Ocadmin Portal 角色（admin.*）

| 角色 name | 中文名稱 | 說明 |
|-----------|----------|------|
| `admin.store_manager` | 門市店長 | 擁有大部分 admin.* 權限 |
| `admin.operator` | 後台操作員 | 日常後台操作 |
| `admin.viewer` | 唯讀檢視者 | 僅擁有 `*.access` 權限 |

### 2.3 跨角色指派

同一帳號可擁有多個角色：

```php
// HR 主管：管理功能 + 自己也是員工
$user->roles: ['hrm.hr_manager', 'hrm.employee']

// 部門主管：ESS + 部門管理
$user->roles: ['hrm.dept_manager', 'hrm.employee']

// 超級管理員：僅限 Ocadmin
$user->roles: ['super_admin']
```

## 三、權限設計

### 3.1 命名規範

**格式：`{portal}.{module}.{resource}.{action}`**

- 固定 **4 段**，使用 **3 個點** 分隔
- 全小寫，snake_case
- resource 一律使用**單數**（如 `product`、`order`、`payment`，不用 `products`）
- module 多層用底線連接

| 段落 | 說明 | 範例 |
|------|------|------|
| `portal` | 系統入口 | `hrm`, `admin` |
| `module` | 功能模組 | `ess`（自助）, `mss`（管理）, `team`（部門主管）, `catalog`, `order` |
| `resource` | 資源名稱 | `profile`, `employee`, `product`, `order`, `role` |
| `action` | 操作動作 | `access`, `modify`, `delete`, `approve`, `export` |

### 3.2 萬用權限（Wildcard）

啟用 Spatie 原生 wildcard permission（`config/permission.php` 中 `enable_wildcard_permission => true`）。

使用 `*` 作為萬用字元：

```
# 單一資源的所有動作
hrm.mss.employee.*        → 符合 hrm.mss.employee.access、.modify、.delete、.export

# 整個模組的所有權限
hrm.mss.*.*               → 符合 hrm.mss.employee.*、hrm.mss.leave.*、hrm.mss.holiday.*

# 整個 Portal 的所有權限
hrm.*.*.*                  → 符合所有 hrm 權限

# 明確指定單一動作
hrm.ess.leave.modify       → 僅符合「請假申請」
```

> **注意**：不要建立 `*.all` 權限，使用 Spatie 原生 wildcard `*` 即可達成相同效果，無需額外的 Gate 邏輯。

### 3.3 命名範例

```
# HRM Portal — ESS 員工自助
hrm.ess.profile.access              個人資料檢視
hrm.ess.profile.modify              個人資料修改
hrm.ess.attendance.access           個人出勤紀錄
hrm.ess.attendance.modify           打卡
hrm.ess.leave.access                個人請假紀錄
hrm.ess.leave.modify                請假申請

# HRM Portal — MSS HR 管理
hrm.mss.employee.access             員工檢視
hrm.mss.employee.modify             員工修改
hrm.mss.employee.delete             員工刪除
hrm.mss.department.access           部門檢視
hrm.mss.department.modify           部門修改

# HRM Portal — Team 部門主管
hrm.team.employee.access            部門成員列表
hrm.team.attendance.access          部門出勤紀錄
hrm.team.leave.approve              部門請假審核

# Admin Portal — 系統管理
admin.system_access.user.access     使用者檢視
admin.system_access.user.modify     使用者修改
admin.system_access.role.access     角色檢視
admin.system_access.role.modify     角色權限設定
admin.config.setting.access         系統設定檢視
admin.config.setting.modify         系統設定修改

# 欄位級權限（action 後加 :column）
hrm.mss.payroll.access:salary       薪資金額
hrm.mss.payroll.access:bonus        獎金金額
```

### 3.4 常用 Action

| Action | 說明 | 涵蓋範圍 |
|--------|------|----------|
| `access` | 檢視 | 列表 + 單筆檢視（原 list + read） |
| `modify` | 修改 | 新增 + 修改（原 create + update） |
| `delete` | 刪除 | |
| `export` | 匯出 | |
| `approve` | 審核 | 適用於請假、出勤、退款等需審核流程 |

> **簡化原則**：能看就能列表（access），能改就能建（modify），刪除與審核獨立控管。

| 資料表 | 說明 |
|--------|------|
| `acl_permissions` | 權限 |

| `acl_role_has_permissions` | 角色-權限關聯 |
| `acl_model_has_permissions` | 使用者-權限關聯（直接指派） |
| `acl_model_has_roles` | 使用者-角色關聯 |

### 4.2 翻譯表

| 資料表 | 說明 |
|--------|------|
| `acl_role_translations` | 角色多語翻譯（display_name, note） |
| `acl_permission_translations` | 權限多語翻譯（display_name, note） |

### 4.3 擴展欄位

`acl_permissions` 額外欄位：

| 欄位 | 類型 | 說明 |
|------|------|------|

| `icon` | varchar, nullable | 圖示 class（如 `fas fa-users`） |

| 欄位 | 類型 | 說明 |

## 五、設定與安裝

### 5.1 config/permission.php（重點摘要）

```php
return [
    'models' => [
        'permission' => App\Models\Acl\Permission::class,  // 自訂 Model
        'role' => App\Models\Acl\Role::class,              // 自訂 Model
    ],

    'table_names' => [
        'roles' => 'acl_roles',
        'permissions' => 'acl_permissions',
        'model_has_permissions' => 'acl_model_has_permissions',
        'model_has_roles' => 'acl_model_has_roles',
        'role_has_permissions' => 'acl_role_has_permissions',
    ],

    'teams' => false,  // 不使用 teams 功能

    'enable_wildcard_permission' => true,  // 啟用萬用權限

    'cache' => [
        'expiration_time' => \DateInterval::createFromDateString('24 hours'),
        'key' => 'spatie.permission.cache',
        'store' => 'default',
    ],

```

### 5.2 自訂 Model

角色和權限 Model 繼承自 Spatie 基礎類別，加上 `HasTranslation` trait 實現多語支援：

```php
// app/Models/Acl/Role.php
class Role extends SpatieRole
{
    use HasTranslation;

    protected $fillable = ['name', 'guard_name', 'sort_order', 'is_active'];
    protected array $translatedAttributes = ['display_name', 'note'];
    protected string $translationModel = RoleTranslation::class;
}

// app/Models/Acl/Permission.php
class Permission extends SpatiePermission
{
    use HasTranslation;

    protected $fillable = [
        'parent_id', 'name', 'guard_name',
        'type', 'icon', 'sort_order', 'is_active',
    ];

    protected array $translatedAttributes = ['display_name', 'note'];
    protected string $translationModel = PermissionTranslation::class;

    // 樹狀結構與選單相關方法見 §8.5
}
```

### 5.3 User Model

```php
// app/Models/User.php
use Spatie\Permission\Traits\HasRoles;

class User extends Authenticatable
{
    use HasRoles;

    /**
     * 檢查使用者是否擁有指定 Portal 的角色
     */
    public function hasPortalRole(string $portal): bool
    {
        if ($this->hasRole('super_admin')) {
            return true;
        }

        return $this->roles->contains(
            fn ($role) => str_starts_with($role->name, $portal . '.')
        );
    }
}
```

## 六、權限檢查

### 6.1 Controller 中

```php
public function index()
{
    // 方式一：手動檢查
    if (!auth()->user()->can('hrm.mss.employee.access')) {
        abort(403);
    }

    // 方式二：authorize（拋出 AuthorizationException）
    $this->authorize('hrm.mss.employee.access');
}
```

### 6.2 Middleware

```php
// 路由中使用
Route::middleware(['permission:hrm.mss.employee.access'])->group(function () {
    Route::get('/employees', [EmployeeController::class, 'index']);
});

// 或
Route::get('/employees', [EmployeeController::class, 'index'])
    ->middleware('can:hrm.mss.employee.access');
```

### 6.3 Inertia + React 前端

後端透過 Inertia 傳遞權限資訊給前端：

```php
// app/Http/Middleware/HandleInertiaRequests.php

public function share(Request $request): array
{
    return [
        ...parent::share($request),
        'auth' => [
            'user' => $request->user(),
            'permissions' => $request->user()?->getAllPermissions()
                ->pluck('name')
                ->toArray() ?? [],
            'roles' => $request->user()?->getRoleNames()->toArray() ?? [],
        ],
    ];
}
```

React 前端使用：

```tsx
// 取得權限資訊
const { permissions } = usePage().props.auth;

// 權限檢查 helper
function can(permission: string): boolean {
    return permissions.includes(permission);
}

// 在 JSX 中使用
{can('hrm.mss.employee.modify') && (
    <Button>新增員工</Button>
)}
```

### 6.4 欄位級權限

```php
// Controller
public function show(Employee $employee)
{
    $data = $employee->toArray();

    // 隱藏無權限的敏感欄位
    if (!auth()->user()->can('hrm.mss.payroll.access:salary')) {
        unset($data['salary']);
    }
    if (!auth()->user()->can('hrm.mss.payroll.access:bonus')) {
        unset($data['bonus']);
    }

    return Inertia::render('Employee/Detail', ['employee' => $data]);
}
```

## 七、資料權限

資料範圍由權限的 module 前綴隱含：

| 權限前綴 | 資料範圍 | 對應角色 |
|----------|---------|---------|
| `hrm.mss.*` | 全公司 | HR 管理角色 |
| `hrm.team.*` | 同部門 | 部門主管 |
| `hrm.ess.*` | 僅自己 | 一般員工 |

### 7.1 Policy

```php
// app/Policies/EmployeePolicy.php

class EmployeePolicy
{
    public function viewAny(User $user): bool
    {
        return $user->can('hrm.mss.employee.access')
            || $user->can('hrm.team.employee.access');
    }

    public function view(User $user, Employee $employee): bool
    {
        // HR 有權限可查看所有
        if ($user->can('hrm.mss.employee.access')) {
            return true;
        }

        // 查看自己
        if ($employee->user_id === $user->id) {
            return true;
        }

        // 部門主管查看下屬
        if ($user->can('hrm.team.employee.access')) {
            return $employee->department_id === $user->employee?->department_id;
        }

        return false;
    }
}
```

### 7.2 請假審核

```php
// app/Policies/LeavePolicy.php

class LeavePolicy
{
    public function approve(User $user, Leave $leave): bool
    {
        // HR 可審核所有
        if ($user->can('hrm.mss.leave.approve')) {
            return true;
        }

        // 部門主管只能審核自己部門的
        if ($user->can('hrm.team.leave.approve')) {
            return $leave->employee->department_id === $user->employee?->department_id;
        }

        return false;
    }
}
```

## 八、選單系統

### 8.1 概念

選單與權限共用同一張 `acl_permissions` 資料表，透過 `type` 欄位區分用途：

| type | 用途 | 層級 | 範例 |
|------|------|------|------|
| `menu` | 選單節點 | L1 ~ L3 | `hrm`, `hrm.mss`, `hrm.mss.employee` |
| `action` | 功能權限 | L4 | `hrm.mss.employee.access`, `hrm.mss.employee.modify` |

`type` 欄位**僅供參考**，Spatie 的 `hasPermissionTo()` / `can()` 只比對 `name`，不看 `type`。

### 8.2 樹狀結構

權限表同時存在 1～4 段的記錄，透過 `parent_id` 構成選單樹：

```
hrm                              (L1, type=menu, parent_id=null)
├── hrm.mss                      (L2, type=menu, parent_id → hrm)
│   ├── hrm.mss.employee         (L3, type=menu, parent_id → hrm.mss)
│   │   ├── hrm.mss.employee.access   (L4, type=action)
│   │   ├── hrm.mss.employee.modify   (L4, type=action)
│   │   └── hrm.mss.employee.delete   (L4, type=action)
│   └── hrm.mss.leave            (L3, type=menu, parent_id → hrm.mss)
│       ├── hrm.mss.leave.access       (L4, type=action)
│       └── hrm.mss.leave.approve      (L4, type=action)
└── hrm.ess                      (L2, type=menu, parent_id → hrm)
    └── hrm.ess.profile          (L3, type=menu, parent_id → hrm.ess)
        ├── hrm.ess.profile.access     (L4, type=action)
        └── hrm.ess.profile.modify     (L4, type=action)
```

> menu 節點（L1～L3）與 action 節點（L4）的 `name` 天然不同（段數不同），不會衝突。

### 8.3 選單產生流程

```
acl_permissions（type=menu）    使用者的權限
        │                           │
        ▼                           ▼
   完整選單樹          ∩        使用者權限集合
        │
        ▼
   使用者可見選單
```

1. 從 `acl_permissions` 撈 `type=menu` → **完整選單樹**
2. 與使用者擁有的權限取交集 → **使用者可見選單**
3. 使用者未擁有的 menu 權限 → 該選單項目隱藏

### 8.4 自動指派父層權限

指派 L4 action 權限時，必須同時指派其 L1～L3 menu 權限，否則選單路徑不完整、使用者看不到入口。

```php
// Role Model
public function assignPermissionWithAncestors(Permission $permission)
{
    // 指派權限本身
    if (!$this->hasPermissionTo($permission)) {
        $this->givePermissionTo($permission);
    }

    // 自動指派所有父層權限（確保選單路徑完整）
    foreach ($permission->ancestors() as $ancestor) {
        if (!$this->hasPermissionTo($ancestor)) {
            $this->givePermissionTo($ancestor);
        }
    }
}
```

範例：指派 `hrm.mss.employee.access` 時，自動一併指派：
- `hrm.mss.employee`（L3, menu）
- `hrm.mss`（L2, menu）
- `hrm`（L1, menu）

### 8.5 Permission Model 擴展

```php
// app/Models/Acl/Permission.php

class Permission extends SpatiePermission
{
    use HasTranslation;

    // 父層權限
    public function parent(): BelongsTo
    {
        return $this->belongsTo(Permission::class, 'parent_id');
    }

    // 子層權限
    public function children(): HasMany
    {
        return $this->hasMany(Permission::class, 'parent_id')
            ->orderBy('sort_order');
    }

    // 取得所有祖先（遞迴向上）
    public function ancestors(): Collection
    {
        $ancestors = collect();
        $parent = $this->parent;
        while ($parent) {
            $ancestors->push($parent);
            $parent = $parent->parent;
        }
        return $ancestors;
    }

    // 是否為選單項目
    public function isMenu(): bool
    {
        return $this->type === 'menu';
    }

    // 是否為功能權限
    public function isAction(): bool
    {
        return $this->type === 'action';
    }
}
```

## 十、多語支援

本系統的角色與權限支援多語翻譯，透過自訂 Model 上的 `HasTranslation` trait 實現。

### 10.1 取得翻譯後的名稱

```php
// 取得當前語系的 display_name
$role->display_name;  // 自動依 app()->getLocale() 回傳

// 指定語系
$role->translate('zh_Hant')->display_name;  // "HR 主管"
$role->translate('en')->display_name;        // "HR Manager"
```

### 10.2 前端顯示

後端傳遞角色/權限資訊時，自動帶入當前語系的翻譯：

```php
// Controller
return Inertia::render('System/Role/Index', [
    'roles' => Role::with('translation')
        ->orderBy('sort_order')
        ->where('is_active', true)
        ->get(),
]);
```

## 十一、總結

### 11.1 快速參考

| 項目 | 格式 | 範例 |
|------|------|------|
| 全域角色 | `{role}` | `super_admin` |
| Portal 角色 | `{portal}.{role}` | `hrm.hr_manager`, `admin.operator` |
| 權限 | `{portal}.{module}.{resource}.{action}` | `hrm.mss.employee.access` |
| 萬用權限 | `{portal}.{module}.{resource}.*` | `hrm.mss.employee.*` |
| 欄位權限 | `{portal}.{module}.{resource}.{action}:{column}` | `hrm.mss.payroll.access:salary` |

### 11.2 檢查方式

```php
// 檢查角色
$user->hasRole('super_admin');
$user->hasRole('hrm.hr_manager');

// 檢查權限（支援 wildcard 比對）
$user->can('hrm.mss.employee.access');

// 檢查 Portal 存取（見 0105_Portal概述.md）
$user->hasPortalRole('hrm');
$user->hasPortalRole('admin');
```
