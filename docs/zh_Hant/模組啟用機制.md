# 模組啟用機制設計

## 概述

參考 OpenCart Extensions 設計一套模組管理機制，讓管理者可以：
1. 檢視所有可用模組
2. 啟用/停用模組
3. 啟用前執行必要檢查
4. 啟用後自動出現在側邊欄

---

## 模組架構

### 目錄結構

```
vendor/elonphp/laravel-ocadmin-modules/
└── src/Modules/                    ← 套件內建模組（不應修改）
    ├── User/
    ├── SystemSetting/
    ├── SystemLog/
    └── ModuleManager/              ← 模組管理器（內建）

app/
└── Ocadmin/
    └── Modules/                    ← 使用者自訂模組
        ├── Product/                ← 商品管理
        ├── Order/                  ← 訂單管理
        └── User/                   ← 可覆蓋套件的 User 模組
```

### 載入順序與覆蓋規則

```
1. 載入套件模組 (vendor/.../src/Modules/)
2. 載入自訂模組 (app/Ocadmin/Modules/)
   ↓
若同名：自訂模組覆蓋套件模組
```

**覆蓋範例：**
- 套件有 `User` 模組，使用者也在 `app/Ocadmin/Modules/User/` 建立同名模組
- 系統使用自訂的 `User` 模組，套件的被忽略
- 這讓使用者可以完全客製化內建功能

### 命名空間

| 來源 | 路徑 | Namespace |
|------|------|-----------|
| 套件 | `vendor/.../src/Modules/User/` | `Elonphp\LaravelOcadminModules\Modules\User` |
| 自訂 | `app/Ocadmin/Modules/Product/` | `App\Ocadmin\Modules\Product` |

## 現有架構分析

### 模組載入流程

```
OcadminServiceProvider::boot()
    ↓
ModuleLoader::loadAll()
    ↓
├── loadStandardModules()  ← 掃描 src/Modules/
└── loadCustomModules()    ← 掃描 app/Ocadmin/Modules/
    ↓
對每個模組執行：
├── loadModuleRoutes()
├── loadModuleViews()
├── loadModuleTranslations()
└── loadModuleMenu()
```

### 目前的啟用機制

- `module.json` 中的 `enabled` 欄位
- `config/ocadmin.php` 中的 `modules` 陣列可覆寫

```php
// config/ocadmin.php
'modules' => [
    'system-log' => true,
    'user' => false,  // 停用
],
```

**問題：** 目前只是靜態配置，沒有 UI 管理介面。

## 設計方案

### 1. 資料庫儲存模組狀態

新增 `ocadmin_modules` 資料表：

```sql
CREATE TABLE ocadmin_modules (
    id BIGINT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
    name VARCHAR(100) NOT NULL COMMENT '模組名稱',
    alias VARCHAR(100) NOT NULL COMMENT '模組別名',
    source ENUM('package', 'custom') DEFAULT 'package' COMMENT '來源',
    version VARCHAR(20) DEFAULT NULL COMMENT '版本號',
    enabled TINYINT(1) DEFAULT 0 COMMENT '是否啟用',
    installed_at TIMESTAMP NULL COMMENT '安裝時間',
    config JSON NULL COMMENT '模組設定',
    created_at TIMESTAMP NULL,
    updated_at TIMESTAMP NULL,

    UNIQUE KEY (alias)
);
```

### 2. 模組管理介面

#### 路由

```
/ocadmin/modules              → 模組列表
/ocadmin/modules/{alias}      → 模組詳情
/ocadmin/modules/{alias}/enable   → 啟用流程
/ocadmin/modules/{alias}/disable  → 停用
```

#### 列表頁面

| 模組 | 版本 | 狀態 | 來源 | 操作 |
|------|------|------|------|------|
| 使用者管理 | 1.0.0 | ✓ 已啟用 | 套件 | [停用] |
| 參數設定 | 1.0.0 | ✗ 未啟用 | 套件 | [啟用] |
| 系統日誌 | 1.0.0 | ✓ 已啟用 | 套件 | [停用] |

### 3. 啟用流程（Enable Flow）

```
使用者點擊 [啟用]
    ↓
Step 1: 讀取 module.json
    ↓
Step 2: 檢查相依性
    - 相依模組是否已啟用？
    - PHP 版本是否符合？
    - Laravel 版本是否符合？
    ↓
Step 3: 檢查資料表
    - migrations 列表中的資料表是否已存在？
    - 若存在，提示：使用現有資料表 / 重新建立（危險）
    ↓
Step 4: 執行 Migrations
    - 若資料表不存在，自動執行
    - 記錄已執行的 migrations
    ↓
Step 5: 執行 Seeders（可選）
    - 詢問是否要匯入預設資料
    ↓
Step 6: 發布資源（可選）
    - Config 檔案
    - Views（供客製化）
    - Assets
    ↓
Step 7: 寫入資料庫
    - ocadmin_modules.enabled = 1
    - installed_at = now()
    ↓
Step 8: 清除快取
    - config:clear
    - route:clear
    - view:clear
    ↓
完成！模組出現在側邊欄
```

### 4. 停用流程（Disable Flow）

```
使用者點擊 [停用]
    ↓
Step 1: 檢查相依性
    - 是否有其他模組依賴此模組？
    - 若有，顯示警告並阻止
    ↓
Step 2: 確認對話框
    - 「確定要停用此模組嗎？資料將保留。」
    ↓
Step 3: 寫入資料庫
    - ocadmin_modules.enabled = 0
    ↓
Step 4: 清除快取
    ↓
完成！模組從側邊欄消失
```

### 5. 解除安裝流程（Uninstall Flow）

```
使用者點擊 [解除安裝]
    ↓
Step 1: 確認對話框（危險操作）
    - 「確定要解除安裝嗎？相關資料表將被刪除！」
    - 需輸入模組名稱確認
    ↓
Step 2: 執行 Rollback Migrations
    ↓
Step 3: 刪除資料庫記錄
    ↓
Step 4: 清除快取
    ↓
完成！
```

## 實作細節

### 修改 ModuleLoader

```php
class ModuleLoader
{
    // 新增：從資料庫讀取啟用狀態
    protected function isModuleEnabled(string $alias): bool
    {
        // 優先從資料庫讀取
        $dbModule = DB::table('ocadmin_modules')
            ->where('alias', $alias)
            ->first();

        if ($dbModule) {
            return (bool) $dbModule->enabled;
        }

        // Fallback 到 config
        return config("ocadmin.modules.{$alias}", true);
    }

    // 修改：載入模組前檢查狀態
    protected function loadModule(string $modulePath, string $moduleName, string $source): void
    {
        $config = $this->getModuleConfig($modulePath);
        $alias = $config['alias'] ?? Str::kebab($moduleName);

        // 檢查是否啟用
        if (!$this->isModuleEnabled($alias)) {
            return; // 跳過此模組
        }

        // 繼續載入...
    }
}
```

## 開發順序

1. **Phase 1: 基礎架構**
   - 建立 `ocadmin_modules` 資料表
   - 修改 `ModuleLoader` 支援資料庫狀態

2. **Phase 2: 列表功能**
   - 掃描所有模組
   - 顯示模組狀態
   - 基本啟用/停用（不含檢查）

3. **Phase 3: 安裝流程**
   - 相依性檢查
   - 資料表檢查
   - Migration 執行
   - Seeder 執行

4. **Phase 4: 進階功能**
   - 資源發布
   - 解除安裝
   - 模組設定頁
