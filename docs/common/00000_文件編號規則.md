# 文件編號規則

本目錄下的文件以 5 位數編號命名，前 2 碼為分類，後 3 碼為序號。

本專案為 **LaravelOcadmin 基礎範例專案（`branchMain`，開源版）**，是所有衍生具體系統（`branchHuabing`、`branchSunline` 等）的基準。本檔同時定義 branchMain 自身與所有衍生 branch 的編號規則。

---

## 第一位數字（命名空間）

| 第一位 | 用途 | 寫入 branch |
|---|---|---|
| `0` | 開發規範後設文件 | branchMain（衍生 branch 不寫此段） |
| `1` | 基礎共用機制 | branchMain |
| `2` | 衍生專案具體實作 | **所有衍生 branch 共用**（branchHuabing、branchSunline、branchAdminlte…） |
| `9` | 議題探討與參考 | branchMain |

衍生 branch 的歸屬靠 git repo 隔離（branchHuabing 寫的 23xxx 與 branchSunline 寫的 23xxx 各自存在於各自 repo），不靠第一位數字區分。

---

## 第二位數字（業務段）

第二位代表業務領域，**跨第一位語意保持一致**：例如 branchMain 的 `13xxx` 是銷售訂單，所有衍生 branch 的 `23xxx` 也是銷售訂單。

| 第二位 | 業務段 | 說明 |
|---|---|---|
| 0 | 開發規範（0x） / 系統全域架構（1x、2x） | 0x 段為 branchMain 開發規範；1x/2x 段為系統全域架構（含參數設定、Settings、認證、選單、日誌等基礎機制） |
| 1 | 基礎主檔 | 組織、公司、部門、行政區、供應商、客戶、會員等跨模組共用主檔 |
| 2 | 商品型錄 | 商品、分類、選項、BOM、計量單位 |
| 3 | 銷售訂單 | 訂單、銷售明細、退貨、發票 |
| 4 | 採購生產（供應鏈） | 採購進貨、應付前流程、生產製造 |
| 5 | 財務會計 | 應付帳款、付款、費用支出、應收／應付分類帳 |
| 6 | 人資差勤 | 員工、出勤、排班、薪資、勞健保 |
| 7 | 報表統計 | 銷售、庫存、會員分析等 |
| 8 | CRM | 客戶服務案件、行銷活動 |
| 9 | 其它與議題探討（1x、2x） / 議題探討與參考（9x） | 雜項、UI、前端、未實作的提案、舊版保留參考 |

---

## 第二位字母擴充（a~z）

業務段 0~9 用滿後，可改用字母擴充第二位（共 36 個段位足夠任何 ERP 規模）：

| 第二位 | 預留語意 |
|---|---|
| `1a` / `2a` | 財務會計細分（如預算、成本中心，若 5x 不敷使用） |
| `1b` / `2b` | 生產製造細分（如製令工單、BOM 計算） |
| `1c~1z` / `2c~2z` | 未來擴充 |

實際啟用字母段時，再於本表登錄具體語意。

---

## 引用慣例

跨 branch 引用編號時加 prefix 避免歧義：

| 寫法 | 適用情境 |
|---|---|
| 裸寫 `10007` | branchMain 內互引、上下文已明確 |
| `branchMain/10007` | 在衍生 branch 文件中引用 branchMain 共用機制 |
| `branchHuabing/23100` | 在 branchMain 或其他 branch 文件中引用 branchHuabing 文件 |

---

## 命名格式

```
{編號}_{標題}.md
```

範例：
- `00000_文件編號規則.md`
- `10000_系統架構.md`
- `10007_權限機制.md`
- `10021_參數設定.md`

---

## 序號配置建議

同一分類下，`x000` 通常為該分類的總覽／概述，`x001` 之後為細節主題；但並非硬性規定，實際依內容重要性與閱讀順序安排。

每段 1000 個序號（x000~x999）足夠用，可按業務子主題彙整。例：

- `13000~13099`：訂單核心
- `13100~13199`：訂單發票
- `13200+`：訂單其他子主題（保留）

---

## 10xxx 主題群內部排列

10xxx 為 branchMain 最大段（含參數設定後共 23 檔），內部依主題群彙整、序號大致遞增：

| 序號範圍 | 主題群 |
|----------|--------|
| 10000 | 總覽 |
| 10001 | Portal |
| 10002–10003 | 國際化（多語、詞彙） |
| 10004–10010 | 認證、安全、帳號（Sanctum、使用者記錄、API 安全、權限、2FA、裝置、全域帳號） |
| 10011 | 選單 UI |
| 10012–10014 | 觀測與錯誤處理（日誌、例外、JSON 回應） |
| 10015 | 非同步與排程（Queue） |
| 10016–10018 | 資料架構（Model 分層、EAV/ZTM、多層選項連動） |
| 10019–10020 | 資源管理（圖片、靜態資源快取） |
| 10021–10022 | 參數設定（Settings 主檔、多層擴充架構） |

---

## 衍生 branch 命名空間

衍生 branch 在 docs 目錄維護自己的編號規則檔，描述本 branch 在 2x 段使用情況（含必要時針對自身的子分段）。例：

- `branchHuabing/docs/md/20000_文件編號規則.md`
- `branchSunline/docs/md/20000_文件編號規則.md`

衍生 branch 的「開發規範後設文件」放在 20xxx 段（系統全域架構），不另設 0x 段。
