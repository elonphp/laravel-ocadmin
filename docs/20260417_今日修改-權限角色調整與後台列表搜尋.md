## 一、角色 ID 區段規劃

| ID 區段 | 用途 | 後台行為 |
|---------|------|---------|
| 1–20 | 系統保留角色 | 不可見、不可改 |
| 21–100 | 系統預設角色 | 可見、可指派 |
| 101+ | 使用者自建角色 | 完全可控（auto_increment 起始） |

### 系統保留角色

| id | name | 說明 |
|----|------|------|
| 1 | `system` | 系統操作標記（service 帳號也賦予此角色） |
| 2 | `developer` | 開發者，Gate::before 無條件放行 |

### 系統預設角色

| id | name | 說明 |
|----|------|------|
| 21 | `super_admin` | Gate::before 放行（`dev_only_permissions` 除外） |

### 移除的角色

- `service` — 不再需要獨立角色，service 帳號改用 `system` 角色
- `demo`、`reader` — 移除，需要時由各專案建立 Portal 角色（如 `admin.reader`）

## 二、Gate::before 分層

```php
Gate::before(function ($user, $ability) {
    if ($user->hasRole('developer')) {
        return true;  // 無條件放行
    }
    if ($user->hasRole('super_admin')) {
        return in_array($ability, config('vars.dev_only_permissions', []))
            ? null   // 走個別綁定
            : true;  // 放行
    }
    return null;
});
```

### 涉及檔案

- [ ] `app/Providers/AppServiceProvider.php` — Gate::before 加入 developer + super_admin 分層
- [ ] `config/vars.php` — 新建，`dev_only_permissions` 預設空陣列
