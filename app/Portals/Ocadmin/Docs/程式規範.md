# 程式規範

## 概述

本文件說明 Ocadmin 的程式開發規範，以「使用者管理」為範例。

---

## 架構原則

### 分層結構

```
Controller → Service → Model
    │           │
    │           └── 業務邏輯（寫入操作）
    └── HTTP 層（查詢、Transaction）
```

### 核心原則

| 原則 | 說明 |
|------|------|
| Controller : Service = 1 : 1 | 每個 Controller 對應一個 Service |
| Transaction | **一律寫在 Controller** |
| Service 職責 | 處理 `create`、`update`、`delete` 或複雜邏輯 |
| 查詢邏輯 | 簡單查詢直接寫在 Controller |

### 何時使用 Service

| Controller 方法 | 規則 | Service 方法 |
|-----------------|------|--------------|
| `store()` | **一律調用 Service** | `create()` |
| `update()` | **一律調用 Service** | `update()` |
| `destroy()` | **一律調用 Service** | `delete()` |
| `getList()` 等查詢 | 直接操作 Model | — |

## View 資料傳遞規範

Controller 傳遞資料到 View 時，**一律使用 `$data` 陣列**：

```php
// ✓ 正確寫法
public function index(Request $request): View
{
    $data = [
        'list' => $this->getList($request),
        'breadcrumbs' => $this->breadcrumbs,
    ];

    return view('user::index', $data);
}

// ✗ 不使用 compact()
public function index(Request $request): View
{
    $list = $this->getList($request);

    return view('user::index', compact('list'));
}
```

**原因：**
- 明確列出傳遞的變數，易於追蹤
- 避免 `compact()` 變數名稱打錯造成的隱性錯誤
- 方便統一加入 `breadcrumbs` 等共用資料

## 麵包屑設定

每個 Controller 需覆寫 `setBreadcrumbs()` 方法，定義該模組的麵包屑路徑：

```php
class UserController extends Controller
{
    public function __construct(
        protected UserService $userService
    ) {
        $this->setBreadcrumbs();
    }

    protected function setBreadcrumbs(): void
    {
        $this->breadcrumbs = [
            (object)[
                'text' => __('ocadmin::common.dashboard'),
                'href' => ocadmin_route('dashboard'),
            ],
            (object)[
                'text' => __('ocadmin::menu.users'),  // 中間節點（選單分類）
                'href' => 'javascript:void(0)',
            ],
            (object)[
                'text' => __('user::user.title'),
                'href' => ocadmin_route('user.index'),
            ],
        ];
    }
}
```

**麵包屑結構：**

| 欄位 | 說明 |
|------|------|
| `text` | 顯示文字（使用語系檔） |
| `href` | 連結網址（中間節點使用 `javascript:void(0)`） |

**View 中使用：**

```blade
<ol class="breadcrumb">
    @foreach($breadcrumbs as $breadcrumb)
        @if($loop->last)
            <li class="breadcrumb-item active">{{ $breadcrumb->text }}</li>
        @else
            <li class="breadcrumb-item"><a href="{{ $breadcrumb->href }}">{{ $breadcrumb->text }}</a></li>
        @endif
    @endforeach
</ol>
```

## Controller 範例

### 完整範例：UserController

```php
<?php

namespace Ocadmin\Modules\User;

use App\Models\Identity\User;
use Illuminate\Http\Request;
use Illuminate\Http\JsonResponse;
use Illuminate\View\View;
use Illuminate\Support\Facades\DB;
use App\Helpers\Classes\OrmHelper;
use Ocadmin\Core\Controllers\Controller;

class UserController extends Controller
{
    public function __construct(
        protected UserService $userService
    ) {
        $this->setBreadcrumbs();
    }

    protected function setBreadcrumbs(): void
    {
        $this->breadcrumbs = [
            (object)[
                'text' => __('ocadmin::common.dashboard'),
                'href' => ocadmin_route('dashboard'),
            ],
            (object)[
                'text' => __('ocadmin::menu.users'),
                'href' => 'javascript:void(0)',
            ],
            (object)[
                'text' => __('user::user.title'),
                'href' => ocadmin_route('user.index'),
            ],
        ];
    }

    /**
     * 列表頁（初始載入）
     */
    public function index(Request $request): View
    {
        $data = [
            'list' => $this->getList($request),
            'breadcrumbs' => $this->breadcrumbs,
        ];

        return view('user::index', $data);
    }

    /**
     * AJAX 入口（列表刷新）
     */
    public function list(Request $request): string
    {
        return $this->getList($request);
    }

    /**
     * 核心查詢邏輯
     */
    protected function getList(Request $request): string
    {
        $query = User::query();
        $filter_data = $request->all();

        // OrmHelper::prepare() 自動處理 filter_*, equal_* 開頭的參數
        OrmHelper::prepare($query, $filter_data);

        // 關鍵字查詢（多欄位 OR 自訂查詢）
        if ($request->filled('search')) {
            $search = $request->search;
            $query->where(function ($q) use ($search) {
                // 使用 OrmHelper::filterOrEqualColumn() 處理萬用字元
                OrmHelper::filterOrEqualColumn($q, 'filter_name', $search);

                $q->orWhere(function ($subQ) use ($search) {
                    OrmHelper::filterOrEqualColumn($subQ, 'filter_email', $search);
                });
            });
        }

        // 預設排序
        $filter_data['sort'] = $request->get('sort', 'id');
        $filter_data['order'] = $request->get('order', 'desc');

        // OrmHelper::getResult() 自動處理排序與分頁
        $users = OrmHelper::getResult($query, $filter_data);

        // 設置分頁器路徑
        $users->withPath(route('lang.ocadmin.user.list'));

        // 建構 URL 參數
        $url = $this->buildUrlParams($request);

        // 準備資料（使用 $data 陣列）
        $data['users'] = $users;
        $data['action'] = route('lang.ocadmin.user.list') . $url;
        $data['url_params'] = $url;

        return view('user::list', $data)->render();
    }

    /**
     * 新增表單
     */
    public function create(): View
    {
        $data = [
            'user' => $this->userService->withDefaults(),
            'breadcrumbs' => $this->breadcrumbs,
        ];

        return view('user::form', $data);
    }

    /**
     * 儲存新資料
     */
    public function store(Request $request): JsonResponse
    {
        $validated = $request->validate([
            'name' => 'required|string|max:255',
            'email' => 'required|email|unique:users,email',
            'password' => 'required|string|min:8|confirmed',
        ]);

        try {
            DB::transaction(function () use ($validated) {
                $this->userService->create($validated);
            });

            return response()->json([
                'success' => true,
                'message' => __('ocadmin::messages.created'),
                'redirect' => ocadmin_route('user.index'),
            ]);
        } catch (\Exception $e) {
            return response()->json([
                'success' => false,
                'message' => __('ocadmin::messages.error'),
            ], 500);
        }
    }

    /**
     * 編輯表單
     */
    public function edit(int $id): View
    {
        $data = [
            'user' => $this->userService->find($id),
            'breadcrumbs' => $this->breadcrumbs,
        ];

        return view('user::form', $data);
    }

    /**
     * 更新資料
     */
    public function update(Request $request, int $id): JsonResponse
    {
        $rules = [
            'name' => 'required|string|max:255',
            'email' => 'required|email|unique:users,email,' . $id,
        ];

        if ($request->filled('password')) {
            $rules['password'] = 'string|min:8|confirmed';
        }

        $validated = $request->validate($rules);

        try {
            DB::transaction(function () use ($id, $validated) {
                $this->userService->update($id, $validated);
            });

            return response()->json([
                'success' => true,
                'message' => __('ocadmin::messages.updated'),
            ]);
        } catch (\Exception $e) {
            return response()->json([
                'success' => false,
                'message' => __('ocadmin::messages.error'),
            ], 500);
        }
    }

    /**
     * 刪除資料
     */
    public function destroy(int $id): JsonResponse
    {
        try {
            DB::transaction(function () use ($id) {
                $this->userService->delete($id);
            });

            return response()->json([
                'success' => true,
                'message' => __('ocadmin::messages.deleted'),
            ]);
        } catch (\Exception $e) {
            return response()->json([
                'success' => false,
                'message' => __('ocadmin::messages.error'),
            ], 500);
        }
    }
}
```

## Service 範例

### 完整範例：UserService

```php
<?php

namespace Ocadmin\Modules\User;

use Illuminate\Support\Facades\Hash;

class UserService
{
    protected string $model;

    public function __construct()
    {
        $this->model = config('ocadmin.models.user', \App\Models\User::class);
    }

    /**
     * 建立新資料
     */
    public function create(array $data): mixed
    {
        $data['password'] = Hash::make($data['password']);

        return $this->model::create($data);
    }

    /**
     * 更新資料
     */
    public function update(int $id, array $data): bool
    {
        $user = $this->model::findOrFail($id);

        if (!empty($data['password'])) {
            $data['password'] = Hash::make($data['password']);
        } else {
            unset($data['password']);
        }

        return $user->update($data);
    }

    /**
     * 刪除資料
     */
    public function delete(int $id): bool
    {
        $user = $this->model::findOrFail($id);

        return $user->delete();
    }

    /**
     * 取得預設值（供新增表單使用）
     */
    public function withDefaults(): mixed
    {
        return new $this->model([
            'name' => '',
            'email' => '',
        ]);
    }
}
```

## 視圖三層架構

### 運作流程

```
1. 使用者點擊「篩選」按鈕
2. JavaScript 收集表單參數，呼叫 /list 路由
3. Controller::list() → getList() 回傳 list.blade.php 的 HTML
4. JavaScript 將 HTML 替換到 #list-container
5. 更新瀏覽器網址列（history.pushState）
```

### 三層分離

| 檔案 | 用途 |
|------|------|
| `index.blade.php` | 主視圖（頁面框架、篩選面板、JavaScript） |
| `list.blade.php` | 表格視圖（資料列表、分頁） |
| `form.blade.php` | 表單視圖（新增/編輯共用） |

---

## 視圖布局規範

### 列表頁布局（index.blade.php）

**桌面版：** 篩選區塊在右側（3 欄），列表在左側（9 欄）

```
┌─────────────────────────────┬───────────────┐
│         使用者列表           │     篩選      │
│         (col-lg-9)          │   (col-lg-3)  │
│                             │   order-last  │
└─────────────────────────────┴───────────────┘
```

**手機版：** 篩選區塊隱藏，顯示 Filter 按鈕切換

```
┌─────────────────────────────────────────────┐
│  標題        [Filter] [+] [刪除]             │
├─────────────────────────────────────────────┤
│              使用者列表                       │
└─────────────────────────────────────────────┘
```

**關鍵 CSS 類別：**

| 類別 | 作用 |
|------|------|
| `order-lg-last` | 桌面版放右側 |
| `d-none d-lg-block` | 手機版隱藏，桌面版顯示 |
| `d-lg-none` | 只在手機版顯示（Filter 按鈕） |

### 表單頁布局（form.blade.php）

**一列一欄位**，使用水平表單（horizontal form）：

```
┌──────────────┬─────────────────────────────────────┐
│   名稱       │  [                                ] │
│  (col-sm-2)  │           (col-sm-10)               │
├──────────────┼─────────────────────────────────────┤
│   Email      │  [                                ] │
├──────────────┼─────────────────────────────────────┤
│   密碼       │  [                                ] │
└──────────────┴─────────────────────────────────────┘
```

**關鍵 CSS 類別：**

| 類別 | 作用 |
|------|------|
| `row mb-3` | 一列一欄位 |
| `col-sm-2 col-form-label` | 標籤（左側） |
| `col-sm-10` | 輸入欄位容器（右側） |
| `required` | 必填欄位標記（加在 row 上） |

## 開發檢查清單

### 模組結構

- [ ] 模組目錄：`Ocadmin/Modules/{ModuleName}/` 或 `Ocadmin/Modules/{Category}/{ModuleName}/`
- [ ] 模組定義：`module.json` 檔案存在
- [ ] 扁平結構：Controller、Service 直接放在模組根目錄
- [ ] 視圖目錄：`views/`
- [ ] 語系目錄：`lang/{locale}/`

### Controller

- [ ] 繼承 `Ocadmin\Core\Controllers\Controller`
- [ ] 注入對應的 Service
- [ ] 覆寫 `setBreadcrumbs()` 設定麵包屑
- [ ] `store()` / `update()` / `destroy()` 調用 Service 並包 `DB::transaction()`
- [ ] `getList()` 處理搜尋篩選與排序
- [ ] View 資料使用 `$data` 陣列傳遞

### Service

- [ ] 實作 `create()`、`update()`、`delete()` 方法
- [ ] 實作 `find()` 方法
- [ ] 實作 `withDefaults()` 方法（供新增表單使用）

### 視圖

- [ ] 三層分離：`index.blade.php` + `list.blade.php` + `form.blade.php`
- [ ] 表單使用 AJAX 提交，顯示 Toast 通知
- [ ] 列表使用 AJAX 刷新，支援分頁與排序
- [ ] 視圖命名空間：`{alias}::viewname`（如 `user::index`）

### 設定檔

- [ ] `module.json`：模組定義
- [ ] `routes.php`：模組路由
- [ ] `menu.php`：側邊欄選單

## OrmHelper 查詢輔助

### 概述

OrmHelper 是 Eloquent 查詢輔助類別，自動處理篩選、排序、分頁。支援三種多語模式：

| 模式 | 常數 | 資料表結構 | 說明 |
|------|------|-----------|------|
| 1 | `TRANSLATION_MODE_SUFFIX` | `xxx_translations` | 翻譯子表（預設） |
| 2 | `TRANSLATION_MODE_EAV` | `xxx_metas` | 純 EAV（暫不實作） |
| 3 | `TRANSLATION_MODE_ZEAV` | `zeav_xxx` | EAV + 快取表 |

### 基本使用

```php
// Controller 中
$query = Product::query();
$params = $request->all();

OrmHelper::prepare($query, $params);  // 自動處理 filter_*, equal_*
$result = OrmHelper::getResult($query, $params);  // 分頁結果
```

### 參數命名規範

| 前綴 | 用途 | 範例 |
|-----|------|------|
| `filter_欄位名` | 彈性查詢（支援萬用字元） | `filter_name=王*` |
| `equal_欄位名` | 精確查詢 | `equal_status=active` |

### ZEAV 模式（translation_mode = 3）

當 Model 設定 `translation_mode = 3` 時，OrmHelper 自動委派給 OrmZeavHelper：

```php
class Product extends Model
{
    use HasZeavTable;

    public $translation_mode = 3;
    public $translation_keys = ['name', 'description'];
}
```

**資料表關係：**
```
products (主表)
├── product_metas (EAV 真實資料)
└── zeav_products (快取表，供查詢用)
```

**自動 JOIN：** 當篩選或排序涉及 `translation_keys` 欄位時，自動 JOIN `zeav_xxx` 表。

詳細說明請參考 `docs/md/OrmHelper.md`。
