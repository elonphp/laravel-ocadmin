# Ocadmin 檔案結構

## 概述

Ocadmin 位於 Laravel 專案的 `app/Portals/Ocadmin/`，使用 Laravel 標準命名空間 `App\Portals\Ocadmin\`。

```
myproject/
├── app/
│   ├── Portals/
│   │   └── Ocadmin/        ← Ocadmin 位置
│   ├── Models/
│   ├── Http/
│   └── ...
├── config/
├── database/
└── ...
```

---

## Composer 設定

Ocadmin 使用 Laravel 的 `App\` 命名空間，不需要額外的 autoload 設定。

```json
{
    "autoload": {
        "psr-4": {
            "App\\": "app/"
        }
    }
}
```

### 安裝步驟

1. 將 Ocadmin 放到 `app/Portals/Ocadmin/`
2. 執行 `composer dump-autoload`
3. 在 `bootstrap/providers.php` 註冊 ServiceProvider：
   ```php
   return [
       App\Providers\AppServiceProvider::class,
       App\Portals\Ocadmin\Providers\OcadminServiceProvider::class,
   ];
   ```

## 命名空間

命名空間使用 `App\Portals\Ocadmin\`：

| 相對路徑 | 命名空間 |
|----------|----------|
| `app/Portals/Ocadmin/Providers/` | `App\Portals\Ocadmin\Providers` |
| `app/Portals/Ocadmin/Http/Controllers/` | `App\Portals\Ocadmin\Http\Controllers` |
| `app/Portals/Ocadmin/Services/` | `App\Portals\Ocadmin\Services` |
| `app/Portals/Ocadmin/ViewComposers/` | `App\Portals\Ocadmin\ViewComposers` |

---

## View 命名空間

視圖使用 `ocadmin::` 作為命名空間：

```php
return view('ocadmin::dashboard.index');
return view('ocadmin::system.setting.index');
return view('ocadmin::system.access.user.form');
```

## 路由機制

路由定義在 `routes/ocadmin.php`，由 `OcadminServiceProvider` 載入。

### 路由名稱範例

| 路由名稱 | URL |
|----------|-----|
| `lang.ocadmin.dashboard` | `/zh-hant/ocadmin` |
| `lang.ocadmin.system.setting.index` | `/zh-hant/ocadmin/system/setting` |
| `lang.ocadmin.system.access.user.index` | `/zh-hant/ocadmin/system/access/user` |

## 未來規劃：Core/Modules 結構

未來可進一步拆分為 Core（框架核心）和 Modules（功能模組）結構：

```
app/Portals/Ocadmin/
├── Core/                            ← 框架核心
│   ├── Console/
│   ├── Http/Controllers/
│   ├── Middleware/
│   ├── Support/
│   ├── Traits/
│   └── ViewComposers/
│
└── Modules/                         ← 功能模組
    ├── Dashboard/
    ├── System/
    └── Account/
```

### Core vs Modules

| 類型 | 說明 |
|------|------|
| **Core** | 框架基礎設施，不含業務邏輯 |
| **Modules** | 可插拔的業務功能模組 |
