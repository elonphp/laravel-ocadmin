# 例外處理機制

## 概述

本文件說明 Laravel Ocadmin 的錯誤處理機制。

錯誤分為兩大類：

| 類型 | HTTP 狀態碼 | 處理方式 | 說明 |
|------|-------------|----------|------|
| **預期的業務邏輯錯誤** | 4xx | `CustomException` | 商品已用於訂單、庫存不足等 |
| **非預期的系統層級錯誤** | 500 | 全域 handler | 資料庫錯誤、程式 bug 等 |

---

## 核心概念

### 預期的業務邏輯錯誤（CustomException）

這類錯誤是**可預期的業務規則限制**，應該給用戶明確的提示訊息。

**範例場景：**
- 刪除商品時，該商品已被訂單引用 → `此商品已用於訂單，無法刪除，請改為停用`
- 庫存不足 → `庫存不足，目前庫存僅剩 5 件`
- 設定鍵重複 → `此設定鍵已存在`

```php
// Service 層主動檢查並拋出
if ($product->orderItems()->exists()) {
    $this->fail('此商品已用於訂單，無法刪除，請改為停用');
}
```

### 非預期的系統層級錯誤（500）

這類錯誤是**程式或環境問題**，錯誤訊息包含敏感資訊（SQL、檔案路徑等），只有工程師能看到完整內容。

**範例場景：**
- 資料表不存在（schema 未同步）
- 欄位名稱打錯
- 資料庫連線失敗
- 外鍵約束失敗（未做業務檢查就直接刪除）

```php
// 不需要特別處理，全域 handler 自動捕捉
$product->delete();  // 若有外鍵約束失敗，自動回傳 500
```

**回應內容：**
- 一般用戶 → `系統發生錯誤，請聯絡管理員。`
- 工程師/debug 模式 → `SQLSTATE[23000]: Cannot delete, foreign key constraint fails...`

## 為什麼需要 CustomException？

### 情境：刪除設定

**沒有業務檢查（不好）：**
```php
public function delete(Setting $setting): void
{
    $setting->delete();  // 直接刪除
}
```
- 若有約束失敗 → 500 錯誤
- 用戶看到：`系統發生錯誤，請聯絡管理員。`（不知道為什麼失敗）

**有業務檢查（好）：**
```php
public function delete(Setting $setting): void
{
    if ($setting->is_system) {
        $this->fail('系統參數無法刪除');  // 400 錯誤
    }

    $setting->delete();
}
```
- 用戶看到：`系統參數無法刪除`（清楚知道原因）

## 全域錯誤處理

檔案位置：`bootstrap/app.php`

### 處理流程

```
Controller / Service
        ↓
    例外發生
        ↓
withExceptions() 捕捉所有例外
        ↓
依例外類型分流處理
        ↓
┌─────────────────────────────────────────────────────────────┐
│ 1. ValidationException  → 422 + 驗證錯誤詳情                 │
│ 2. HttpException        → 依狀態碼 + 自訂訊息                │
│ 3. CustomException      → 依狀態碼 + sendJsonErrorResponse  │
│ 4. 其他（含資料庫錯誤）  → 500 + sendJsonErrorResponse       │
└─────────────────────────────────────────────────────────────┘
        ↓
    JSON Response
```

### 各類例外處理說明

| 順序 | 例外類型 | HTTP 狀態碼 | 處理方式 | 來源 |
|------|----------|-------------|----------|------|
| 1 | `ValidationException` | 422 | 直接回傳驗證錯誤 | FormRequest 驗證失敗 |
| 2 | `HttpException` | 依例外 | 直接回傳訊息 | `abort(404, '...')` |
| 3 | `CustomException` | 依例外 | `sendJsonErrorResponse` | Service 層業務檢查 |
| 4 | 其他 | 500 | `sendJsonErrorResponse` | 資料庫錯誤、程式 bug |

## CustomException 類別

檔案位置：`app/Exceptions/CustomException.php`

```php
namespace App\Exceptions;

class CustomException extends Exception
{
    public function __construct(
        string $generalError = '系統發生錯誤，請聯絡管理員。',
        ?string $sysError = null,
        int $statusCode = 500,
        ?\Throwable $previous = null
    );

    public function getGeneralError(): string;
    public function getSysError(): ?string;
    public function getStatusCode(): int;
}
```

| 參數 | 說明 | 範例 |
|------|------|------|
| `generalError` | 給一般用戶看的友善訊息 | `'此設定已被使用，無法刪除'` |
| `sysError` | 給工程師看的詳細訊息（可選） | `'Setting 123 is referenced by...'` |
| `statusCode` | HTTP 狀態碼 | `400`, `404`, `403` |

## BaseService 輔助方法

檔案位置：`app/Services/BaseService.php`

Service 繼承 `BaseService` 後，可使用以下方法：

```php
use App\Services\BaseService;

class SettingService extends BaseService
{
    public function delete(Setting $setting): void
    {
        // 業務錯誤（預設 400）
        if ($setting->is_system) {
            $this->fail('系統參數無法刪除');
        }

        // 帶詳細訊息
        if ($setting->isInUse()) {
            $this->fail(
                '此設定正在使用中，無法刪除',
                "Setting {$setting->id} is used by module X"
            );
        }

        $setting->delete();
    }
}
```

### 可用方法

| 方法 | HTTP 狀態碼 | 用途 |
|------|-------------|------|
| `$this->fail($msg)` | 400 | 業務邏輯錯誤 |
| `$this->fail($msg, $sysMsg, 409)` | 自訂 | 帶狀態碼的錯誤 |
| `$this->notFound($msg)` | 404 | 找不到資源 |
| `$this->forbidden($msg)` | 403 | 權限不足 |
| `$this->unavailable($msg)` | 503 | 服務不可用 |
| `$this->error($msg)` | 500 | 系統錯誤 |

## sendJsonErrorResponse()

這是一個輔助函式，用於根據環境和角色決定回傳哪種訊息。

### 參數說明

| 參數 | 說明 | 範例 |
|------|------|------|
| `$generalError` | 給一般用戶看的友善訊息 | `'系統發生錯誤，請聯絡管理員。'` |
| `$sysError` | 給工程師看的詳細訊息 | `'SQLSTATE[42S02]: Table not found...'` |
| `$statusCode` | HTTP 狀態碼 | `400`, `500` |
| `$request` | 當前請求（用於取得登入用戶） | - |

### 訊息顯示邏輯

| 條件 | 顯示內容 |
|------|----------|
| `APP_DEBUG=true` | `$sysError`（詳細錯誤） |
| 用戶有 `ocadmin.sys_admin` 角色 | `$sysError`（詳細錯誤） |
| 其他情況 | `$generalError`（友善訊息） |

## 各元件關係

| 項目 | withExceptions | CustomException | sendJsonErrorResponse |
|------|----------------|-----------------|----------------------|
| **是什麼** | 全域錯誤處理器 | 自訂例外類別 | 輔助函式 |
| **位置** | `bootstrap/app.php` | `app/Exceptions/` | `bootstrap/app.php` |
| **職責** | 捕捉所有例外並分流 | 攜帶業務錯誤資訊 | 根據角色決定回傳訊息 |
| **何時觸發** | 任何例外發生時 | Service 主動拋出 | 被 withExceptions 呼叫 |

### 關係圖

```
Service 層
    │
    ├── 業務檢查失敗 ──→ throw CustomException ──┐
    │                                           │
    └── 其他操作（可能失敗）                      │
            │                                   │
            ├── 資料庫錯誤 ──→ QueryException ───┤
            │                                   │
            └── 其他錯誤 ───────────────────────┤
                                                │
                                                ▼
                                    withExceptions() 捕捉
                                                │
                    ┌───────────────────────────┼───────────────────────────┐
                    │                           │                           │
                    ▼                           ▼                           ▼
            CustomException              QueryException              其他例外
            (預期的業務錯誤)             (資料庫錯誤)               (非預期錯誤)
                    │                           │                           │
                    └───────────────────────────┴───────────────────────────┘
                                                │
                                                ▼
                                    sendJsonErrorResponse()
                                                │
                                    根據 debug/角色 決定訊息
                                                │
                                                ▼
                                          JSON Response
```

## 使用時機對照表

| 情境 | 處理方式 | 說明 |
|------|----------|------|
| 表單欄位驗證 | `FormRequest` 或 `$request->validate()` | Laravel 自動回 422 |
| 資源不存在 | `findOrFail()` 或 Route Model Binding | 自動回 404 |
| 業務規則限制 | `CustomException` | 給用戶明確提示 |
| 第三方 API 錯誤 | `CustomException` | 隱藏 API 錯誤細節 |
| 資料庫/程式錯誤 | 不處理（全域捕捉） | 自動回 500 |

## 實際範例

### Service 使用範例

```php
namespace App\Portals\Ocadmin\Modules\System\Setting;

use App\Models\System\Setting;
use App\Services\BaseService;

class SettingService extends BaseService
{
    public function create(array $data): Setting
    {
        $data['locale'] = $data['locale'] ?? '';
        return Setting::create($data);
    }

    public function update(Setting $setting, array $data): Setting
    {
        $data['locale'] = $data['locale'] ?? '';
        $setting->update($data);
        return $setting;
    }

    public function delete(Setting $setting): void
    {
        // 業務規則檢查
        if ($setting->code === 'system' && $setting->key === 'core') {
            $this->fail('核心系統參數無法刪除');
        }

        $setting->delete();
    }

    public function batchDelete(array $ids): int
    {
        // 檢查是否包含系統參數
        $systemCount = Setting::whereIn('id', $ids)
            ->where('code', 'system')
            ->where('key', 'core')
            ->count();

        if ($systemCount > 0) {
            $this->fail('選取的項目包含核心系統參數，無法刪除');
        }

        return Setting::whereIn('id', $ids)->delete();
    }

    public function exists(string $locale, string $code, string $key, ?int $excludeId = null): bool
    {
        $query = Setting::where('locale', $locale)
            ->where('code', $code)
            ->where('key', $key);

        if ($excludeId) {
            $query->where('id', '!=', $excludeId);
        }

        return $query->exists();
    }
}
```

## 不需要 CustomException 的情況

簡單 CRUD 不需要 Service 和 CustomException：

```php
// Controller 直接處理
public function update(Request $request, Setting $setting)
{
    $validated = $request->validate([...]);  // 驗證失敗自動 422
    $setting->update($validated);
    return redirect()->back()->with('success', '更新成功');
}

public function edit(Setting $setting)  // Route Model Binding 自動 404
{
    return view('setting::form', compact('setting'));
}
```

## API 回應格式

### 驗證錯誤（422）
```json
{
    "success": false,
    "message": "資料驗證失敗",
    "errors": {
        "code": ["命名空間為必填"],
        "key": ["此設定鍵已存在"]
    }
}
```

### 業務錯誤（4xx）
```json
{
    "success": false,
    "message": "此設定正在使用中，無法刪除"
}
```

### 系統錯誤（500）
```json
// 一般用戶
{
    "success": false,
    "message": "系統發生錯誤，請聯絡管理員。"
}

// debug 模式 / sys_admin
{
    "success": false,
    "message": "SQLSTATE[23000]: Cannot delete or update a parent row..."
}
```
